## 设计方案

### 方案选择: B — 激进优化 (Aggressive Push)
**理由**: 目标300%+需要显著提升，保守微调因子权重难以弥补114%的差距。

### 具体改动

#### 1. OUTPUT_FILE
- 从 `backtest_momentum.csv` → `/home/pebynn/quant/evo_optimizer/backtest_A.csv`

#### 2. TOP_N
- 从8 → 10（范围8-10上限，扩大候选池增加分散度）

#### 3. 新因子: turnover_ratio, amount_ratio
- **turnover_ratio**: 成交额 / 20日均成交额 (类似vol_ratio但用成交额，捕捉不同信号维度)
- **amount_ratio**: 20日成交额变化率 = (amount_t / amount_t-20) - 1，即20日成交额动量
- 计算方式: 在compute_factors()中，已有amount数据可用
- 加入FACTOR_NAMES和FACTOR_WEIGHTS

#### 4. 因子权重优化
重新分配权重，强调长期动量和新增因子：
```python
FACTOR_WEIGHTS = {
    'ret_60d_z': 0.30,      # 长期动量主信号
    'ret_20d_z': 0.22,      # 中期动量
    'ret_5d_z': 0.03,       # 降低短期噪声权重
    'vol_ratio_z': 0.05,    # 量比
    'rsi_14_z': 0.04,       # RSI
    'boll_pos_z': 0.03,     # 布林位置
    'atr14_pct_z': 0.04,    # 波动率
    'turnover_ratio_z': 0.05,  # 新增：成交额量比
    'amount_ratio_z': 0.04,    # 新增：成交额动量
}
```
总权重: 0.80，归一化后自动分配。

#### 5. 分批止盈 (Batch Take-Profit)
**机制**: 每个头寸分2个tranche，各50%仓位。
- **TP1**: 浮盈达到+30%时，卖出Tranche1（50%仓位）
- **TP2**: 剩余仓位浮盈达到+50%时，卖出Tranche2（剩余仓位）
- **实现方式**: positions字典增加字段 `tranche` ('full'|'half') 和 `tp1_hit` (bool)
- 已触发TP1的仓位标记tranche='half'，后续只剩一半仓位参与NAV计算
- NA V计算调整: 半仓的贡献按0.5计算

- [x] LEVERAGE=1.0 不可变 ✓
- [x] 因子用T-1数据（已有xsec_rank用前一天截面）✓
- [x] 入场用次日开盘价 ✓
- [x] OUTPUT_FILE指向backtest_A.csv
- [x] 数据源: fallback_resolver.load_kline() ✓

### 替代方案
- **方案A (保守)**: 仅调因子权重+TOP_N=8, 不加新因子和分批止盈。风险低但提升有限。
- **方案C (极激进)**: TOP_N=12, 杠杆1.5。违反铁律LEVERAGE=1.0，拒绝。

### 风险
1. **新因子共线性**: turnover_ratio与vol_ratio可能高度相关(r>0.8)，导致冗余
2. **分批止盈复杂度**: 半仓跟踪增加代码复杂度，需仔细处理NAV计算
3. **TOP_N=10增加换手**: 更多持仓可能导致交易成本上升
4. **过拟合风险**: 权重调整基于单一市场期间(2025-05→2026-04)
