# R3-A 代码修复计划: 消除4个未来函数偏差

## 任务概述
修复 strategy_v2.py 中 OOS 验证发现的4个 CRITICAL 未来函数偏差。

## OOS 发现的4个偏差

### B1: LOOK_AHEAD_MOMENTUM_ENTRY (行132-149)
**问题**: `check_momentum_entry()` 使用当日收盘价判断是否以当日开盘价买入。
- 收盘价在开盘后才能知道 → 典型未来函数
- 原公式: `ret_5d = (close - open) / open`

**修复方案**: 改用前一日收盘价计算5日收益
- 在 `compute_factors_v2()` 中新增 `ret_5d` 列 (5日收益率, 基于收盘价)
- `check_momentum_entry()` 改为接受前一日 ret_5d 值
- 买入段传入 `xr.loc[c, 'ret_5d']` (昨日5日收益)

### B2: LOOK_AHEAD_TRAILING_EXIT (行330-336, 362-369)
**问题**: 移动止盈触发后使用当日开盘价作为退出价。
- A股无法在当日开盘价卖出（触发条件为当日低点，开盘价在前）
- 同样影响 hard stop 的隐含退出假设

**修复方案**: 延迟退出机制
- 当 stop/trail 基于当日盘中价格触发 → 标记为 pending_exit
- 次日在开盘时执行退出 (使用次日开盘价)
- 不影响恢复: 若次日停牌/无数据 → 使用最近收盘价

### B3: T_PLUS_1_VIOLATION (行354-379)
**问题**: 策略假设同日买卖可行，A股T+1禁止日内卖出。
- 买入段(行382-389)在同一天可能买入，卖出段(行354-379)可能立即卖出

**修复方案**: 添加 T+1 约束
- 检查 `pos[c]["in"]` (入场日期) 是否等于当前日期 `cd`
- 若相等 → 跳过该持仓的卖出检查
- 但保留 stop/trail 的 pending 标记 (可以标记，在次日执行)

### B4: NO_SLIPPAGE (行322-329)
**问题**: 未考虑涨跌停和滑点，止损10%可能在跌停板无法平仓。

**修复方案**: 添加涨跌停感知
- 检查是否跌停: `low <= prev_close * 0.9` (主板10%跌停)
- 跌停日无法执行止损 → pending 到次日
- 添加滑点成本: 止损退出时额外扣除0.5%滑点
- 一般交易已含 TC_COST_V2=0.001

## 代码变更清单

### 1. compute_factors_v2() — 新增 ret_5d
- 添加: `ret_5d = _pct_chg(close, 5)`
- 输出: `r["ret_5d"] = ret_5d`

### 2. check_momentum_entry() — 重构签名
- 旧: `check_momentum_entry(xt_row, entry_price, threshold)`
- 新: `check_momentum_entry(ret_5d_prev, threshold)`
- 逻辑: `return ret_5d_prev >= threshold - 1e-8`

### 3. 主回测循环 — 新增 pending_exits 机制
- 每日开始时处理 pending_exits (使用当日开盘价执行退出)
- 当 stop/trail 触发时 → 添加 pending_exit 而非立即执行
- 从 pos 中移除 (不参与当日 NAV)

### 4. 主回测循环 — T+1 检查
- 卖出段: 跳过 `pos[c]["in"] == cd` 的持仓
- 若持仓被 pending_exit 跳过, 保留到次日

### 5. 主回测循环 — 涨跌停检测
- 新增 `is_limit_down(xt_row, prev_close)` 辅助函数
- 止损触发 + 跌停 → pending_exit (不立即执行)

### 6. 退出价格修正
- 调仓退出 (drop): 仍用当日开盘价 ✓ (基于昨日信号)
- 止损/移动止盈: 改用次日开盘价 (通过 pending_exits)
- Jan/Apr清仓: 现有用收盘价 → 改为次日开盘价

## 验证计划
- 运行 2024-06~2024-12 小样本回测 (约120交易日)
- 确保无语法错误
- 检查退出原因分布是否合理
- T+1: 验证无当日买卖对

## 风险
- 延迟退出可能导致额外损失 (次日低开) — 这是必须接受的现实成本
- ret_5d 新因子可能轻微改变排名 — 仅在 entry filter 使用，不影响排名
- 涨跌停检测简化 (仅用10%阈值) — 对创业板/科创板不够准确，但主策略以主板为主
