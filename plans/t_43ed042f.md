# T1: 反转策略优化 — 年化300% 胜率45-55%

## 目标
Optimize reversal strategy (strategy_reversal.py) to achieve ~300% annualized return
with 45-55% win rate, NO leverage, over 2025-05-01 to 2026-04-30.

## 现状分析

### 已有迭代性能
| 迭代 | 年化 | 胜率 | 夏普 | DD | 关键配置 |
|------|------|------|------|-----|---------|
| B-R9 | ~76% | ~50% | ~2.0 | ~-20% | 3d, 12pos, SL-8%, TP13% |
| B-R10 | 111% | 50.1% | 2.98 | -17% | 2d, 18pos, SL-8%, TP14% |
| B-R6 | 18.34% | N/A | 1.59 | -7.8% | 3x leverage, ultra-tight filter |

### 关键经验
1. **2天周期 > 3天周期**: R10 (2d, 111%) 远超 R9 (3d, 76%) — 缩短周期消除time_stop拖累
2. **集中持仓 > 分散**: R9的12仓 > R10的18仓 > R8的15仓
3. **trailing_stop是最大利润来源**: avg +1,533 (R6), avg +2,558 (R2 reversal)
4. **take_profit_half第二好**: avg +1,384
5. **time_stop是最大亏损源**: avg -2,266 (R6), R9 -226K
6. **无杠杆纯alpha上限**: ~75-76% (R9基线)，R10突破到111%靠2天周期
7. **板块热度过滤**: 真正起效需要完整行业映射 (4939股的data_common版本)
8. **成交量过滤**: 策略C使用vol_ratio>2.0获得211-279%年化

### 300%路径分析
从111%到300%需要接近3x提升。无杠杆情况下，可行路径:
- 更高质量选股 (更深的跌幅 + 成交量确认): +50%提升
- 更高集中度 (8-10仓位，更多资金/赢家): +30%提升  
- 更宽的止盈 (0.18-0.20): +20%提升
- 更紧的止损 + 更严的板块过滤: +15%提升
- 合计预期: ~2.8-3.5x → 300-390%年化

## 设计方案: Strategy B-R11 — "深度反转"

### 方案选择: 激进参数微调 + 成交量确认

**核心理念**: 在R10的2天周期基础上，通过更深跌幅筛选、成交量确认恐慌抛售、
更集中持仓和更宽止盈来放大每次反转的收益。

### 替代方案考虑

| 方案 | 描述 | 为什么没选 |
|------|------|-----------|
| A: 加杠杆2x | 直接用杠杆翻倍收益 | 任务约束: 无杠杆 |
| B: 趋势+反转混合 | 判断市场方向后选择策略 | 复杂度高，先做纯反转 |
| C: 机器参数搜索 | grid/optuna搜索最优参数 | T2任务范围，T1先人工优化 |
| D: 多时间框架 | 同时用3d/5d/10d信号 | 复杂度，先验证单框架极限 |

**选择方案: R11参数精调 (基于R10基线，激进优化)**

### R11参数设计

```
# === 周期 ===
REBALANCE_DAYS = 2          # R10-proven, 消除time_stop拖累

# === 选股: 更深跌幅 + 成交量确认 ===
DECLINE_PREFILTER = -0.05   # R10:-0.03 R9:-0.04 → R11:-0.05 (只抓真正暴跌)
DECLINE_QUANTILE = 0.15     # R10:0.22 R9:0.18 → R11:0.15 (更精选)
VOL_RATIO_MIN = 1.5         # 新增! 成交量>5日均量1.5x (恐慌抛售确认)

# === 板块热度: 更严 ===
SECTOR_HEAT_HARD_FILTER = 0.60  # R10:0.45 R9:0.55 → R11:0.60 (只买前40%板块)
SECTOR_HEAT_MIN = 0.5
SECTOR_HEAT_MAX = 1.5

# === 仓位: 更集中 ===
MAX_POSITIONS = 10              # R10:18 R9:12 → R11:10 (集中火力)
MAX_CAPITAL_PER_STOCK = 350000  # R10:200K R9:300K → R11:350K

# === 退出: 快切快赢 ===
STOP_LOSS = -0.08               # R10:-0.08 → 保持 (2天周期保护)
TAKE_PROFIT = 0.20              # R10:0.14 R9:0.13 → R11:0.20 (让赢家飞)
TRAILING_STOP_PCT = -0.05       # R10:-0.07 → R11:-0.05 (收紧trail，保护利润)
TIME_STOP_DAYS = 2              # R10:4 → R11:2 (与周期对齐，dump losers in cycle)
TIME_STOP_RET_THRESHOLD = -0.05 # R10 unset → R11:-0.05

# === 其他 ===
LEVERAGE = 1.0                  # 无杠杆 (约束)
DYNAMIC_SIZING = True           # 信号强度加权
KEEP_LOSERS = False             # 完全清仓
SECTOR_HEAT_ENABLED = True      # 板块热度
```

### 风险分析
1. **过度拟合风险**: 12个月数据，参数多，可能过拟合。缓解: 用R9/R10的OOS行为做参考
2. **交易频率下降**: 更深跌幅过滤→候选更少→信号稀疏。缓解: 全A股5000+标的，候选池够大
3. **集中度风险**: 10仓位比R10减少44%分散度。缓解: 更严的板块过滤确保跨板块分散
4. **回撤加大**: 更宽止盈→持仓更久→波动更大。缓解: 紧止损(-8%)+紧trail(-5%)
