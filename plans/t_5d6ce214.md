## 1. BRAINSTORMING — Design Decisions

### 1.1 Problem Summary
R3 achieved -62.81% annual return. Four root causes identified:
1. No market direction filter → reversal catches falling knives in bear markets
2. Sector heat only covers 20/5350 stocks → effectively neutral (sector_boost=1.0)
3. Entry at open price → buys at potentially bad intraday prices
4. No dynamic position sizing → full exposure even in weak markets

### 1.2 R4 Improvement Design

#### 1.2.1 Market Timing Filter (PRIMARY FIX)
**Problem**: Reversal strategy must avoid buying during sustained downtrends.
**Design decision**: Use universe-derived market proxy (equal-weighted avg of all stocks' close) since we don't have index data (MySQL inaccessible, Sina parquet = 100 stocks only).

**Implementation**:
- Compute MARKET_PROXY_RET = equal-weighted mean of ret_1d across all stocks each day
- Compute MA20 and MA60 of market proxy cumulative return
- Entry gate: only allow buys when MA20 > MA60 (golden cross = uptrend) OR ADX(14) > 25
- New param: `MARKET_TIMING_ENABLED = True`
- New param: `MARKET_TIMING_MODE = "ma_cross"` (options: "ma_cross", "adx", "both")

**Alternative considered**: Loading external index data (000001.SH from a file) — rejected because no data source available.
**Risk**: Equal-weighted proxy may not perfectly represent market direction, but it's a reasonable approximation and strictly better than no filter.

#### 1.2.2 Sector Mapping Expansion
**Problem**: daily_signal_detail covers only ~20 stocks, making sector_boost=1.0 for 99.6% of stocks.

**Design decision**: Augment with stock-code-based rough sector classification.
- 000xxx/001xxx/002xxx → Shenzhen main board (manufacturing-heavy)
- 300xxx → ChiNext (tech/growth)
- 600xxx/601xxx/603xxx/605xxx → Shanghai main board (traditional)
- 688xxx → STAR (tech/innovation)
- Others → "other"

Then compute sector heat for these 5 broad groups instead of relying on daily_signal_detail.

**Implementation**:
- New function: `_classify_broad_sector(code)` → returns one of 5 broad sectors
- Modify `compute_sector_heat` to accept a fallback mapping function
- When MySQL industry_map covers <200 stocks, augment with broad sector classification
- New param: `BROAD_SECTOR_ENABLED = True`

**Alternative considered**: Loading Shenwan industry index data from files — rejected, no data source.
**Risk**: Broad sector classification is coarse (5 groups vs 28 Shenwan industries) but covers 100% of stocks vs 0.4%.

#### 1.2.3 Entry Timing Optimization
**Problem**: Current execution buys at open price on rebalance day. This means:
- Signals are computed on day T using day T-1 lagged factors
- Buy executes at day T open — same as the signal evaluation price
- No intraday slippage consideration

**Design decision**: Add a 1-day execution delay (close confirmation pattern).
- On rebalance day T: compute signals using day T's data (all lagged by 1 day)
- Execute entries at day T+1 open (if T+1 exists)
- This introduces 1-day signal-to-execution gap, which is conservative and realistic

**Implementation**:
- Add `pending_entries` list that stores signals for next-day execution
- At each day: first execute pending entries from yesterday, then compute new signals, store for tomorrow
- New param: `ENTRY_DELAY = True` (enable next-day execution)

**Alternative considered**: Using close price for entry — rejected, close prices can have anomalies.
**Risk**: Reduces trade frequency slightly (lose 1 day), but improves realism and avoids buying at the same price used for signal.

#### 1.2.4 Dynamic Position Sizing
**Problem**: Full position sizing in weak markets amplifies drawdowns.

**Design decision**: Scale position size based on market proxy trend strength.
- When market proxy MA20 > MA60: normal sizing (1.0x)
- When market proxy MA20 <= MA60: reduced sizing (0.5x)
- When market proxy is in severe downtrend (close < MA60 for 5+ days): minimal sizing (0.3x)

**Implementation**:
- Compute market trend strength score (0.0–1.0) based on:
  - MA20/MA60 ratio mapped to [0.3, 1.0]
  - Cap at 1.0 for strong uptrends
- Multiply position capital allocation by trend strength score
- New param: `DYNAMIC_SIZING_MARKET = True`

**Alternative considered**: Using VIX-like volatility scaling — rejected, too complex and untested.
**Risk**: Reduces returns in weak-but-recovering markets, but should significantly reduce drawdowns.
