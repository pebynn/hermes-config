## 设计方案

### 方案选择: 无依赖纯标准库解析 + JSON快照diff

**方案A (选定)**: Python stdlib only — `urllib.request` 拉取 README, `re` 解析 markdown bullet 条目,
`json` 存储/加载快照, `sys.exit(0)` 静默退出。

**替代方案B (拒绝)**: 使用 `github3.py` / `PyGithub` — 增加依赖，cron 环境可能不可用。
**替代方案C (拒绝)**: 使用 BeautifulSoup / markdown parser — 同样是额外依赖，README 结构简单，正则足够。

### 风险
- GitHub raw 可能被墙/超时（如本次curl用了30s+）。对策：设置合理超时 + 连接重试。
- README 格式变更导致解析失败。对策：解析失败时输出错误到 stderr，cron 会收到异常报告。
- 首次运行无快照文件。对策：视为全量新条目，生成 baseline 快照但不报告（静默）。

### 核心逻辑
1. 拉取 `https://raw.githubusercontent.com/0xNyk/awesome-hermes-agent/main/README.md`
2. 正则提取所有项目条目：`[name](url) by [author](url) - description`
3. 加载上次快照 `~/.hermes/scripts/awesome_hermes_last_scan.json`
4. Diff: new_entries = current - last, removed = last - current
5. 若无新条目且无移除: 输出空 → 静默（no_agent cron 语义）
6. 若有变化: 输出 Markdown 报告到 stdout + 写入文件 + 更新快照
