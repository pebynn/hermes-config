## 一、设计方案

### 1.1 问题诊断 (R1-B 回顾)

| 指标 | R1-B 值 | 问题 |
|:-----|:--------|:-----|
| 年化回报 | 62-99% | 距300%目标差200+pp |
| 交易笔数 | 2051 | 过多 (≈8笔/天)，信号噪声大 |
| 胜率 | 51.2% | 尚可但盈利幅度不足 |
| 最大回撤 | -24.16% | 可接受 |

**根因**: 入场门槛太低 (ret_5d < -3% + bottom 18%)，导致大量弱信号入场，噪音交易稀释收益。

### 1.2 R2 改进方向与技术方案

#### 改进1: 提高信号阈值 — z-score替代简单百分比

**方案选择**: 使用 ret_5d 的滚动 z-score (60日窗口) 作为主信号，替代简单的 ret_5d < -3%

```
z_ret5d = (ret_5d - μ_60d) / σ_60d
入选条件: z_ret5d < -2.0  (相当于2σ外，原方案≈1σ)
```

**替代方案**: 使用分位数收紧 (DECLINE_QUANTILE: 0.18 → 0.08)
- 更简单但无法适应波动率变化
- **选择 z-score** 因为它能自适应市场波动率（高波动时放宽、低波动时收紧）

**风险**: z-score 计算需要 60 天回看窗口，warmup 必须 ≥ 60 天。当前 WARMUP_DAYS=130 足够。

#### 改进2: 增加持仓过滤 — 流动性与市值

**方案选择**: 
- 流动性过滤: avg_daily_amount (20日滚动均值) >= 50M CNY (可用 kline.成交额)
- 市值过滤: 尝试从 MySQL 加载市值数据，失败则降级为仅流动性过滤

```
当日可用列: code, open, close, high, low, volume, amount (成交额)
avg_amount_20d = rolling(20).mean() of amount
过滤: avg_amount_20d >= 50_000_000 (5000万)
```

**替代方案**: 
- 用 `volume * close` 近似成交额 — 不够精确，且 amount 列已存在
- 使用静态市值文件 — 增加维护成本

**风险**: 市值数据在 kline 表中不存在，需额外查询。如 MySQL 不可用，市值过滤降级。

#### 改进3: 优化仓位分配 — 信号强度动态加权

**方案选择**: 三级信号分级
```
强信号: z_ret5d < -3.0 → weight=2.0
中信合: -3.0 ≤ z_ret5d < -2.0 → weight=1.0
弱信号: z_ret5d ≥ -2.0 → 不入选 (已在改进1过滤)

最终 weight 归一化: weight / sum(weights) → 仓位比例
最小仓位: 1% (clip lower=0.01)
最大仓位: 3% (clip upper=0.03, 风险控制)
```

**替代方案**: 
- 连续权重函数 (如 sigmoid) — 引入额外超参，过度拟合风险
- **选择分级** 因为更可解释、可调试，且 R2 阶段不需要过度复杂化

**风险**: 分级断点 (-2.0/-3.0) 可能不是最优。后续 R3 可由 optimizer 调整。

### 1.5 新增因子计算

```python
# z-score of ret_5d
df["ret_5d_ma60"] = df.groupby("code")["ret_5d_raw"].transform(
    lambda s: s.rolling(60, min_periods=30).mean())
df["ret_5d_std60"] = df.groupby("code")["ret_5d_raw"].transform(
    lambda s: s.rolling(60, min_periods=30).std())
df["z_ret5d_raw"] = (df["ret_5d_raw"] - df["ret_5d_ma60"]) / df["ret_5d_std60"].replace(0, 1e-8)
df["z_ret5d"] = df.groupby("code")["z_ret5d_raw"].shift(1)  # lag 1d

# avg daily amount (20d rolling)
df["amount_ma20_raw"] = df.groupby("code")["amount"].transform(
    lambda s: s.rolling(20, min_periods=10).mean())
df["avg_amount_20d"] = df.groupby("code")["amount_ma20_raw"].shift(1)  # lag 1d
```
