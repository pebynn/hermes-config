## 2. Design Decisions

### Chosen Combination: A + B (止损优化 + 止盈升级)

**Direction A — Stop-Loss Optimization**:
- **A1**: STOP_LOSS from -5% → **-7%** (wider, reduces false stops)
- **A2**: Add **ATR dynamic stop**: when pnl < -atr_multiple * (ATR/entry_price), trigger
  - `ATR_STOP_MULTIPLE = 1.5` — if stock drops >1.5x ATR from entry, exit
  - This is adaptive: volatile stocks get wider stops, calm stocks tighter

**Direction B — Take-Profit Upgrade**:
- **B1**: TAKE_PROFIT from 0.15 → **0.25** (+25% full exit)
- **B2**: PARTIAL_TP from 0.08 → **0.12** (+12% partial lock-in at 50%)

**Direction C — NOT implemented (kept for future R8-C)**:
- Would add "连续2日收阳线确认" filter
- Deferred because win_rate already near target (44.7%); reducing false signals could hurt trade count

### Implementation Strategy

All new params as module-level constants, clearly marked `# R7-C`.

Existing constants that change:
- `STOP_LOSS = -0.05` → `STOP_LOSS = -0.07`  # R7-C: widened from -5% to reduce false stops
- `TAKE_PROFIT = 0.15` → `TAKE_PROFIT = 0.25`  # R7-C: upgrade from +15% to +25%
- `PARTIAL_TP = 0.08` → `PARTIAL_TP = 0.12`   # R7-C: upgrade from +8% to +12%

New constants:
- `ATR_STOP_MULTIPLE = 1.5`  # R7-C: ATR dynamic stop-loss multiplier
- `ATR_STOP_ENABLED = True`   # R7-C: enable/disable toggle

New functionality:
- In `check_entry_conditions`: no change (entry conditions unchanged)
- In exit logic: add ATR dynamic stop as additional trigger alongside fixed -7% stop
  - ATR stop: `pnl < -ATR_STOP_MULTIPLE * (atr_at_entry / entry_price)`
  - The ATR value at entry time must be stored in position dict

Output file: `OUTPUT_FILE = ".../backtest_C_R7_v1.csv"`

### Alternative Considered

- **Direction C alone**: Adding 2-day confirmation would reduce trade count (already low at 450/year in A-share)
- **A+C**: 止盈 stays conservative, winners don't run enough
- **B alone**: Stop-loss rate stays high, small losses keep accumulating
- **A+B (chosen)**: Balances loss reduction + winner potential — best expected outcome

### Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Wider stop (-7%) increases avg loss | Medium | Medium | ATR dynamic stop caps max loss adaptively |
| Higher TP (+25%) reduces hit rate | Medium | Low | Partial TP at +12% still locks gains |
| ATR stop computation overhead | Low | Low | Pre-computed ATR, just lookup at entry |
| Backtest slower (ATR lookup) | Low | Low | O(1) lookup via precomputed array |
