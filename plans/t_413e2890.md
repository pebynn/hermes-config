```
factors → composite_rank() → scores
```

### 1.3 PolicyWeightEngine 核心设计

```
PolicyWeightEngine
├── __init__(cache_dir, halflife_days, halflife_major, max_decay_days)
├── apply(scores, cur_date, stock_industry_map) → Series
│   ├── market_mult = _get_market_multiplier(date)     # CCTV宏观
│   ├── industry_signals = _load_active_signals(date)   # EastMoney行业
│   └── for each stock: weight = market_mult × industry_mult (decayed)
├── _get_market_multiplier(date) → float               # 读cctv/{date}.json
├── _load_active_signals(date) → Dict{industry:mult}   # 读{date}.json + 衰减
├── _calc_effective_multiplier(data, days_since) → float
├── _score_to_multiplier(score) → float                 # 查表映射
├── _direction_to_multiplier(ind_data) → float          # net_ratio计算
└── precompute_signals(date_list) → Dict                # 批量预计算（回测用）
```

### 1.4 关键技术决策

| 决策点 | 选择 | 理由 |
|:-------|:-----|:-----|
| 数据缓存格式 | JSON文件按日期 | 与policy_quant.py输出一致 |
| 衰减模型 | max(0, 1-t/h)² | 线性平方衰减，温和尾部 |
| 宏观×行业 | 乘法叠加 | 两信号独立，叠加合理 |
| 行业映射缺失 | 落回宏观乘数 | 不强制要求完整映射 |
| 回退机制 | try/except → ×1.0 | 不透传异常，安静降级 |
| 初始化时机 | 构造函数中条件创建 | policy_weight_enabled=True时 |
| 注入点数量 | 2处(L306调仓 + L367窗口) | 确保两个选股路径一致 |

### 1.5 风险与缓解

| 风险 | 缓解 |
|:-----|:-----|
| 历史信号缓存不存在 | precompute_signals()失败 → 打印WARN → 所有乘数=1.0 |
| AKShare网络抖动 | 缓存TTL机制 + try/except兜底 |
| 行业映射不全 | 无映射→macromult，不报错 |
| 乘数范围过宽 | 默认×0.80~×1.20，可通过配置收紧 |
| 回测不可重复 | 离线缓存保证确定性 |
