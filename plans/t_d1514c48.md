### 2.2 Data Pipeline
```
1. Load Sina kline → compute returns, turnover, volatility
2. Load financial cache → extract EPS sequences per stock
3. Load fund flow cache → build date-indexed main_net lookup
4. Load share DB → compute market cap = close × shares
5. Filter universe: market_cap ≥ 30B, avg_volume ≥ 30M
6. Detect events: latest financial report date + est. 30d lag
7. Compute pseudo-SUE: QoQ EPS surprise / historical EPS std
8. Apply SUE filter: > 1.5
9. Confirm fund flow: 3-day post-event main_net > 0
10. Filter pre-event drift: 30d return < 20%
11. Enter positions: equal weight, max 10, max 15%/stock
12. Hold: 15-20 trading days
13. Exit: hold expiry OR 3d consecutive main_net < 0
```

## 3. Strategy Design

### 3.1 Event Detection (Adaptation)
- **Input**: Financial cache `date` column (quarter-end: 20250331, 20250630, etc.)
- **Estimated announcement**: report_date + 30 calendar days
- **Event universe**: stocks with report_date + 30 ∈ [backtest_start - 30, backtest_end]
- **Rationale**: A-share financial reports are due within 4 months of period end (annual), 1 month (quarterly). Using 30d as estimate.

### 3.2 Pseudo-SUE Computation
- **Actual EPS**: latest quarter's `eps` from financial cache
- **Expected EPS**: prior quarter's `eps` from financial cache  
- **Sigma**: standard deviation of QoQ EPS changes over last 8 quarters
- **SUE = (EPS_current - EPS_prior) / sigma**
- **Filter**: SUE > 1.5
- **Limitation**: This is QoQ momentum, not true analyst surprise. Noted in plan.

### 3.3 Fund Flow Confirmation
- **Window**: 3 trading days post event_date
- **Metric**: cumsum(main_net) over window > 0
- **Source**: fund_flow_{date}.parquet
- **Fallback**: If fund_flow date not found, skip confirmation (pass)

### 3.4 Universe Filters
1. **Market cap ≥ 30B**: close × shares from share_db, computed on event date
2. **Daily turnover ≥ 30M**: avg(amount) over 20 trading days pre-event
3. **Pre-event 30d return < 20%**: (close_event / close_{-30} - 1) < 0.20
4. **Kline data available**: stock must be in Sina parquet (100 stocks)
5. **Financial data available**: must have ≥ 2 quarters of EPS

### 3.5 Position Management
- **Entry**: On event_date + fund_flow confirmation
- **Weight**: equal weight, max 15% per stock
- **Max positions**: 10
- **Hold period**: 15-20 trading days (randomized per stock in range)
- **Exit triggers**:
  - Hold period expired (primary)
  - 3 consecutive days of main_net < 0 (fund flow exit)
  - Delisted / data stops

### 3.6 Backtest Engine
- Follow ReversalBacktest class pattern from strategy_reversal.py
- Custom engine, NOT backtrader (no backtest_engine.py exists)
- Track: trades[], daily_nav[], portfolio{}
- Metrics: annual return, Sharpe, max drawdown, win rate, monthly returns

## 8. Design Decisions & Tradeoffs

### Decision 1: Custom engine vs backtrader
- **Choice**: Custom engine (following ReversalBacktest pattern)
- **Rationale**: No shared/backtest_engine.py exists; existing strategies use custom engines; simpler to debug
- **Alternative**: Install backtrader and build Strategy subclass — more standard but overengineered for this data

### Decision 2: Pseudo-SUE vs skip SUE
- **Choice**: Compute pseudo-SUE from QoQ EPS changes
- **Rationale**: Task requires SUE; analyst forecasts unavailable; QoQ surprise is the closest proxy
- **Risk**: May not capture true earnings surprise effect; correlation with real SUE unknown

### Decision 3: Announcement date estimation
- **Choice**: financial report period end + 30 calendar days
- **Rationale**: Chinese quarterly reports due within 1 month of quarter end
- **Risk**: Actual dates vary; some companies delay; adds noise to event study

### Decision 4: Fund flow fallback
- **Choice**: Skip confirmation if fund_flow data missing for a date
- **Rationale**: Only 5 days of fund flow data cached; don't want to exclude events due to missing data
- **Risk**: Weakens the "confirmation" aspect of the strategy

## 9. Risks
1. **Limited universe**: Only 100 stocks in Sina parquet → may have few events meeting all criteria
2. **No true SUE**: Pseudo-SUE may not capture earnings surprise signal
3. **Fund flow coverage**: Only 5 days cached, may miss most events
4. **No out-of-sample**: Single parquet file, can't do walk-forward validation
5. **Financial data timing**: Report period end dates ≠ announcement dates
