# 策略C — 政策概念+资金流融合 实施计划

## 一、设计方案

### 1.1 架构选择

**方案A: 独立策略脚本（现有模式）** → 参考 strategy_reversal.py / strategy_momentum.py
- 优点: 快速实现，现有代码可复用
- 缺点: 不符合任务规格（要求继承 shared/backtest_engine.py）
- **不采用**

**方案B: 分层架构（选择）** → shared/ 提供基础层 + strategy.py 继承
```
shared/
  data_loader.py    → 数据加载 (K线/资金流/概念成分)
  factor_lib.py     → 因子计算 (概念强度/资金流评分)
  risk_manager.py   → 风控 (个股止损/组合止损/熔断)
  backtest_engine.py → 回测基类 (日循环/NAV/交易记录)

strategy.py → 继承 BacktestEngine，实现策略逻辑
```

### 1.2 数据源选择

| 数据 | 来源 | 理由 |
|------|------|------|
| K线日数据 | MySQL stock_kline / Sina parquet | 现有基础设施 |
| 概念板块K线 | MCP get_concept_kline | 获取概念历史表现 |
| 概念成分股 | MCP get_concept_constituents | 获取受益股名单 |
| 资金流向 | EastMoney API (precache模式) + MCP get_fund_flow | 主力资金确认 |
| 市值过滤 | data_common.get_stock_list + 总股本×收盘价 | 50-400亿过滤 |

### 1.3 策略流程（回测模拟）

**月度信号生成（模拟Step1+2）:**
1. 获取全量概念列表 (MCP: get_concept_list)
2. 对每个概念计算近1月收益率 (MCP: get_concept_kline)
3. 按概念动量排名，取Top 2-3概念
4. 对每个Top概念，获取成分股 (MCP: get_concept_constituents)
5. 市值过滤 50-400亿，选取3-5只核心受益股
6. 合并去重 → 候选池

**每日确认（Step3）:**
1. 检查候选池中每只股票的主力资金净流入
2. 剔除连续3日主力净流出的股票
3. 新增持续流入（3日净流入>0）的股票

**风控（Step3.5）:**
1. 个股止损 -8%: 当日最低价触及→次日开盘平仓
2. 组合止损 -15%: 当日组合回撤超15%→全平，空仓3日
3. 月度调仓: 每月初重新选股

### 1.4 替代方案及不采用理由

| 替代方案 | 不采用理由 |
|----------|------------|
| web_search实时选概念 | 回测无法自动化web_search，用概念动量替代 |
| 实时MCP调用每只股票 | 性能开销大(5000+次调用)，回测需预缓存 |
| backtrader框架 | 现有代码均不用backtrader，引入复杂度高，且内存模式不匹配 |
| 按市值加权 | 任务要求等权持有 |

### 1.5 风险

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 概念动量≠政策强度 | 🔴 | 回测结论需标注"概念动量代理"，实盘中需人工判别政策 |
| MCP调用超时 | 🟡 | 使用缓存+批量调用，回测时预加载 |
| 资金流数据覆盖不全 | 🟡 | EastMoney仅覆盖活跃股票(~600-1500只)，未覆盖股票默认为中性 |
| 月频调仓导致过少交易 | 🟡 | 回测期12个月，预计60-120笔交易，统计显著性不足 |

## 三、技术细节

### NAV计算（重要教训）
- 不使用entry_price直接加总（会导致NAV爆炸）
- 使用prev_close追踪：
  ```python
  day_ret = sum(cur_c / prev_close for each position) / MAX_POSITIONS - 1
  nav *= (1 + day_ret * LEVERAGE)
  ```

### 回测日循环
```
for each trading day:
    1. 检查是否为月初 → 触发月度选股
    2. 检查持仓: stop_loss / portfolio_stop
    3. 资金流确认: 剔除连续3日流出, 新增持续流入
    4. 执行买入/卖出 (次日开盘价)
    5. 计算当日NAV
```

### MCP调用策略
- 回测期间概念列表预加载一次
- 概念K线批量缓存到parquet
- 资金流数据使用EastMoney API（比MCP更快）
