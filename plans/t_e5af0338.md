## 1. 设计方案

### 1.1 基线分析 (R9 on Sina parquet, 2025-05-01→2026-04-30)

| 指标 | R9基线 |
|:--|:--|
| 年化收益 | **-3.23%** |
| 总收益 | -4.57% |
| 夏普 | -0.26 |
| 最大回撤 | -17.30% |
| 胜率 | 53.4% |
| 交易数 | 205 buys / 279 sells |

**核心问题诊断**:
1. **候选池严重不足**: 仅100只股票, DECLINE_PREFILTER=-4% + DECLINE_QUANTILE=0.18导致大多数调仓日无信号(205 buys / 242 days = 0.85/d)
2. **止损吞噬利润**: stop_loss平均-1,444 (75笔), 是最大亏损源
3. **时间止损杀伤力大**: time_stop平均-7,004 (7笔)
4. **止盈半仓也是负的**: take_profit_half平均-219 (77笔) — 说明止盈后剩余仓位被止损或时间止损吃掉
5. **板块热度不可用**: Sina模式下无industry_map, SECTOR_HEAT_ENABLED被强制关闭
6. **调仓清仓是唯一赚钱的退出**: rebalance_close_full平均+1,117 (118笔)

### 1.2 方案选择: R11 — 宽池+集中持仓+宽松止损

**设计理念**: 在100只股票的小宇宙中, 必须大幅放宽选股条件增加候选密度, 同时通过更集中的持仓和更宽松的止损, 让反转头寸有足够时间发育。

#### 参数变更 (R9 → R11)

| 参数 | R9 | R11 | 理由 |
|:--|:--|:--|:--|
| START_DATE | "2026-01-01" ❌ | "2025-05-01" ✅ | **任务约束: 不可修改** |
| LEVERAGE | 1.0 | 1.0 | **任务约束: 不可修改** |
| DECLINE_PREFILTER | -0.04 | **-0.02** | 宽进: 100股池需要更多候选 |
| DECLINE_QUANTILE | 0.18 | **0.35** | 宽进: 底部35%而非18% |
| MAX_POSITIONS | 12 | **8** | 集中持仓提高单票影响力 |
| REBALANCE_DAYS | 3 | **2** | 更快调仓, 更多机会 |
| STOP_LOSS | -0.08 | **-0.12** | 给反转更多呼吸空间 |
| TAKE_PROFIT | 0.13 | **0.18** | 更高目标, 让赢家跑 |
| TRAILING_STOP_PCT | -0.06 | **-0.04** | 更紧的移动止损保护利润 |
| TIME_STOP_DAYS | 3 | **5** | 更长的持有期 |
| TIME_STOP_RET_THRESHOLD | -0.03 | **-0.06** | 仅深度下跌才触发时间止损 |
| MAX_CAPITAL_PER_STOCK | 300,000 | **500,000** | 更高集中度 |
| DYNAMIC_SIZING | True | True | 保留(有效) |
| SECTOR_HEAT_ENABLED | True→sina下false | False | sina下无条件关闭 |
| SECTOR_HEAT_HARD_FILTER | 0.55 | 0.55 | sina下不生效, 不变 |
| KEEP_LOSERS | True | False | 调仓时丢弃亏损股(需代码实现) |
| TREND_SCORE_ENABLED | False | False | 不加趋势过滤 |

#### 关键代码改动(非参数):
1. **实现KEEP_LOSERS=False**: 当前代码调仓时无条件清仓, KEEP_LOSERS参数未使用。需要在调仓逻辑中加入保留/丢弃判断。
2. **实现HOLD_WINNERS**: 调仓时保留盈利>WINNER_THRESHOLD的持仓, 只卖亏损的。
