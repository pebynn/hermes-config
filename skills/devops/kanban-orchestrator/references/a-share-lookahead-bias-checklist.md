# A股回测未来函数偏差检测清单

> 来源: 策略A(主力资金+动量增强) OOS验证发现4类偏差，短周期年化414.4%→全周期73.46%，16.7x虚高。
> 检测时机: 任何策略年化>100%或胜率>60%或夏普>3.0时强制执行。

## 4类必检偏差

### 1. LOOK_AHEAD_MOMENTUM_ENTRY
**症状**: 策略用"当日收盘价"判断是否以"当日开盘价"买入
**错误**: 收盘价在开盘后才产生，开盘时无法知道当日收盘价
**检测**: grep 买入逻辑附近的 `close` / `收盘` — 因子计算是否使用同日数据
**修复**: 因子用T-1日数据，入场用T日开盘价

### 2. LOOK_AHEAD_TRAILING_EXIT
**症状**: 日内触发移动止盈、使用当日开盘价作为退出价
**错误**: A股T+1禁止同日卖出，且开盘价出现在触发条件之前（低点可能在开盘后）
**检测**: 检查 trailing_stop / 移动止盈 退出价是否为同日数据
**修复**: 退出价用次日开盘价（T+1约束）

### 3. T_PLUS_1_VIOLATION
**症状**: 策略假设同日可买卖（day trading）
**错误**: A股T+1制度禁止日内卖出
**检测**: 检查是否存在 buy_date == sell_date 的交易
**修复**: 买入后最早次日卖出，加入 `sell_idx >= buy_idx + 1`

### 4. NO_SLIPPAGE
**症状**: 未考虑涨跌停限制和滑点
**错误**: 止损10%在跌停板可能无法平仓，实盘执行价≠回测假设价
**检测**: 检查是否有涨跌停板跳过逻辑；回测退出价是否直接使用触发价
**修复**: 加入涨跌停检测（连续跌停无法平仓），加入滑点折扣(0.1-0.3%)

## 快速诊断SQL（针对kanban.db）

```sql
-- 检测可疑高收益任务
SELECT id, title, assignee, result 
FROM tasks 
WHERE status='done' 
  AND (result LIKE '%年化%1__%' OR result LIKE '%annual%1__%')
  AND title LIKE '%回测%'
ORDER BY created_at DESC;
```

## 检测脚本

```bash
# 在策略输出目录执行
grep -n 'close\|收盘\|open\|开盘' strategy_*.py | grep -i 'buy\|entry\|enter'
grep -n 'buy_date.*sell_date\|entry.*exit' backtest_*.py
```

## 真实案例

**A策略R2 (2026-05-13)**:
- IS回测(4个月): 年化414.4%, 胜率88.0%, 夏普9.38 — 触发检测
- OOS-A(2024-2025): 年化343.1% — 异常一致（同为偏差产物）
- OOS-B(2025全年): 年化466.5% — 异常一致
- 发现: 4类偏差全部命中 → verdict=FAIL
- 全周期(5年)真实年化: 73.46%, 胜率27.8%, 夏普2.08

**核心教训**: IS与OOS一致高回报≠策略有效，可能是同一偏差在不同时段复现。偏差模式下IS/OOS会虚假一致。
