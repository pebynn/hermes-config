## 代码层：Tier2 聚类检测

`detect_ai_clusters(text, tier1_words, max_cluster_distance=3)` — 滑动窗口检测相邻段落中Tier1词集群。

- 扫描连续3段（窗口大小可调），累计命中Tier1词数
- ≥3个不同Tier1词同时出现在相邻段落 → 标记为"AI段落集群"
- 返回警告列表，含命中词详情

## 代码层：Tier3 密度检测

`ai_density_score(text, tier1_words, threshold_per_1k=2.5)` — 全文Tier1词密度监控。

- 统计清洗后文本中残留Tier1词总次数，换算为每千字密度
- 超过2.5次/千字 → 警告，列出高频残留词
- 阈值可按文章类型调整

## 发布前审核守门员 (audit_guard.py)

`audit_guard.py` 是发布前的最后一道自动化防线，独立运行于 `publish_draft.py` 管线之前。三重检测：

### 1. 合规检测 (BLOCK级 — 一个触发即阻止发布)
- 个股买卖建议：触发词(买入/卖出/推荐/目标价等) + 6位股票代码检测 (`(?<!\d)\d{6}(?!\d)`)
- "建议"违禁词：兜底全量扫描
- 违规承诺：保证/稳赚/必涨/内幕/翻倍等
- AIGC标识：文章必须包含 "AI辅助创作"/"AIGC" 等标识

### 2. 数据准确性检测 (WARN级)
- 成交额/涨跌幅/点位/家数 vs `all_data.json` 交叉验证
- 含个股涨停涨幅、资金流绝对值匹配
- 技术分析点位(支撑/压力/整数关口)自动排除
- AI编造数字检测：扫描文章中所有数字，与原始数据对比

### 3. AI味检测 (WARN级)
- 110+残留AI词扫描（复用Tier1词表）
- Em dash密度 (>2/千字=WARN)
- 粗体密度 (>5处=WARN)
- 三连排比句检测

**用法**:
```bash
python3 audit_guard.py --date 2026-05-07
python3 audit_guard.py --file ~/writing-data/drafts/2026-05-06-每日复盘.md
python3 audit_guard.py --date 2026-05-07 --json  # JSON输出
```

**退出码**: 0=PASS, 1=WARN, 2=BLOCK

**集成到发布管线**:
```python
from audit_guard import audit_draft
result = audit_draft(date_str="2026-05-06")
if result["risk_level"] == "BLOCK":
    sys.exit(2)  # 阻止发布
```

`publish_draft.py` 已集成：审核→PASS自动发布，WARN默认继续(可用`--strict`阻止)，BLOCK阻止。
