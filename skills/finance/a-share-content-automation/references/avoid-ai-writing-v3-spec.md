# avoid-ai-writing + content-creator v3 完整集成规范

## 强制执行规则

所有 daily/weekly 图文内容生成后必须经过三大后处理管线，禁止跳过。

## 1. Prompt 层（生成前预防）

System prompt 嵌入以下规则（已应用到 `generate_review.py` + `weekly_summary.py`）：

### 🔴 禁用词（45+，出现即AI味）

**虚词类**：值得注意的是、需要指出的是、毫无疑问、毋庸置疑、显然、可谓、堪称、一出、可谓是、从历史经验看、是积极信号、值得关注的是、还需要关注的是

**财经AI高频**：强势反弹、强势领跑、大举扫货、完整爆发、全线飘红、显著回暖、成为全场焦点、赚钱效应、情绪面、资金面、基本面回暖、量能方面、资金进场意愿、做多情绪、绝对主线、明星、情绪过于一致

**句式类**：不仅...更、在...背景下、这是...的信号、三连排比句、Em dash（—）超过2处、粗体超过3处

### ✅ 替代写法
不做"强势反弹"→写"涨了X%，收复Y点"
不做"资金面"→直接写"主力净买入X亿"
不做"赚钱效应"→写"涨停X家，跌停X家"

## 2. 代码层（Tier1-3清洗）

### Tier1：直接替换（45词）
`scrub_ai_vocabulary()` 函数，每日复盘+周总结脚本均内嵌。45词自动替换。

### Tier2：正则模式清除
- `在...背景下` → 删除
- `这是...的信号` → 删除
- `不仅仅...更` → 重构

### Tier3：密度检测
- Em dash 密度 > 2.2/1000字 → 警告
- 粗体 > 5处 → 警告

## 3. 后处理层（第二遍审计 + '建议'违禁词）

```python
# 0. 建议违禁词清除（复合短语优先→句式替换→兜底裸词删除）
# 1. AI词汇清洗（Tier1+正则）
# 2. 结构性检测（EmDash/粗体）
# 3. 第二遍审计（Tier1残留扫描）
# 4. SEO评分
# 5. 内容验证
```

### 建议违禁词替换表

| 匹配模式 | 替换 |
|:--|:--|
| 仓位管理建议 | 仓位管理思路 |
| 操作建议 | 关注方向 |
| 投资建议 | 投资参考 |
| 仓位建议 | 仓位管理 |
| 止盈止损建议 | 止盈止损参考 |
| 建议关注 | 可关注 |
| 建议等 | 可等 |
| 残留裸"建议" | 直接删除 |

## 4. 动态亮点评分（标题生成）

`_compute_highlights()` — 5维度评分：
- 指数波动（科创>3%、创业>2%、上证>1.5%）
- 主力资金异常（>200亿重点、>100亿关注）
- 成交额异常（>3万亿爆发、<8000亿冰点）
- 涨停广度（>100只强势）
- 板块分化（领涨板块>4%）

排序后取最高分作为标题核心。Prompt 注入"今日最亮眼"段，AI据此动态生成标题。

## 5. content-creator SEO

- 动态关键词：从当日板块数据提取（非硬编码"科创50"）
- 前100字点题：必须包含当日最核心数据
- 标题评分：长度20-30字 + 含数字 + 含关键词
- 自然融入关键词，不堆砌
