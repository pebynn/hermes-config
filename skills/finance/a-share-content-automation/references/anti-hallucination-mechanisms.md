# 防AI幻觉机制 — A股复盘管线

## 问题背景

DeepSeek / GPT 等 LLM 在撰写复盘文章时，常虚构数字（成交额、涨跌幅、家数等），尤其是在"非精确时间点"指标（如"本周累计"、"合计"、"近5日"）上。用户反馈可信度低。

## 5道防线

### 防线1：采集层预计算（collect_data.py）

AI **不擅长计算**。所有可能需要AI自行相加/相减/求和的派生指标，都在采集阶段算好写入JSON。

| 字段 | 预计算逻辑 | 作用 |
|:----|:----------|:-----|
| `market.total_turnover` | 上证成交额 + 深证成交额 | 避免AI自行加总各指数 |
| `data_date` | Pandas DataFrame索引或"日期"列 | 标注北向资金等延迟数据的实际日期 |
| 涨跌比、涨停跌停比 | 采集时已计算 | 避免AI用"家数相除"操作 |

### 防线2：Prompt注入精确值（generate_review.py / weekly_summary.py）

所有数字在注入Prompt前已经是精确小数，提示词不包含"约"、"近"、"超"等模糊词。

**错误做法**：
```
两市合计成交额约{total_turnover:.0f}亿
```

**正确做法**：
```
两市合计成交额: {total_turnover:.0f}亿
```

### 防线3：System-Level Anti-Hallucination指令

每条Prompt的约束区必须包含此指令（中英文）：

```
⚠️ 数字准确性（关键！）：以上所有数据中的金额（成交额、资金净流入等）、涨跌幅、家数均为精确数值，AI只需直接使用这些数据撰写文章。严禁自行计算、估算或编造任何数字。文章中出现的每一个数据（包括但不限于成交额、涨跌幅、家数、点位）必须能在上文数据中找到对应来源。
```

### 防线4：数据时效性标注

延迟数据（如北向资金有2-3天滞后）必须标注实际数据日期。

Prompt数据区输出格式：
```
北向资金净流入XX亿（最新数据日期: 2026-04-30）
```

这样AI即使在Prompt中看到「今日」也会按实际日期描述。

### 防线5：降级方案

AI API不可用时不硬撑。`generate_fallback_report()` 和 `generate_fallback_weekly()` 直接使用数据模板拼接文本，完全不含AI生成内容。

## 验证方式

生成完成后，用以下正则扫描文章中的数字是否能在对应数据JSON中找到：

```bash
# 从文章中提取所有数字/百分比
grep -oP '\d+\.?\d*' draft.md | head -20

# 与 all_data.json 中的值交叉验证
python3 -c "
import json
d = json.load(open('all_data.json'))
# 检查成交额是否一致
print(d['market']['total_turnover'])
"
```
