# 拼多多中老年女装日常运营清单

> **平台层认知**：先加载 `pdd-platform-mechanics` 理解拼多多活动驱动机制，再执行本清单。活动是最大流量入口。

## 课前自检 — 每日先确认管线是否活着

在执行每日清单之前，先跑3分钟快速自检。这能防止你在一个已经停滞的系统中做无用功。

```bash
# 从 PDD 运营目录抓取最新状态
echo "=== 订单 ==="
ls -lt ~/PDD/运营/orders/ | head -3
echo "=== 退货 ==="
ls -lt ~/PDD/运营/returns/ | head -3
echo "=== 评价 ==="
ls -lt ~/PDD/运营/reviews/ | head -3
echo "=== 库存 ==="
ls -lt ~/PDD/运营/inventory/ | head -3
```

**状态解读**:
- 文件日期距今 >3天 → 数据管线停了，先恢复数据拉取再做事
- 所有订单 status=pending → 订单处理停滞，需先处理积压
- 所有差评 reply=null → 差评拦截未执行，需先回复
- 库存 sold_7d=0 → 库存数据过期，需同步

**恢复优先级**:
1. P0: 订单处理 (待发货 → 标记发货)
2. P0: 差评回复 (黄金2小时已过, 尽量补救)
3. P1: 退货处理 (pending 审批)
4. P1: 库存同步 (重跑库存采集)
5. P2: 选品重启 (新商品采集)

## 每日必做（30分钟）

按顺序执行：

### 1. 活动版位检查（P0优先）
- 检查所有商品是否有有效活动在跑（秒杀/9.9特卖/领券中心/百亿补贴等）
- 即将到期的活动提前3天续报或更换活动
- 记录各活动位的流量和转化数据（评估活动ROI）
- 重点关注：**活动是拼多多最大免费流量来源**（占40-50%），无活动=自断流量

### 2. 订单处理
- 查看昨日订单量，异常波动排查
- 处理待发货订单，确保当日17点前发出
- 设置发货提醒，避免漏发

### 3. 客服响应
- 回复所有未读消息，目标30秒内响应
- 常用快捷短语：`您好亲` / `请稍等` / `感谢您的支持`
- 关注7天无理由退换和运费险相关咨询

### 4. 库存检查
- 检查商品库存，缺货款及时下架补货
- 确保爆款SKU不断货（特别是加大码）

### 5. 竞品动态 + 平台公告
- 浏览TOP3竞品最新变化：价格/评价/主图
- 查看拼多多商家后台公告（政策变化/新规/活动招商）
- 特别关注：商保会动态、仅退款规则调整、自动跟价变化

### 6. 退货/差评拦截（黄金2小时）
- 查看新产生的差评/中评
- 2小时内主动联系客户
- 话术模板：`您好，非常抱歉给您带来不好的体验！方便告诉我们具体哪里不满意吗？我们立刻为您处理。`
- 处理分级：
  - 小问题（线头/小瑕疵）→ 补偿3-5元红包
  - 中等问题（尺码偏差）→ 退货退款包邮+优惠券
  - 大问题（严重质量/发错货）→ 全额退款不退货，真诚道歉
  - 差评撤销率约25-30%

### 7. 活动版位检查（新增v8.0）
- 检查已报名活动的审核状态（通过/驳回/待审）
- 查看各活动版位资源位是否正常展示
- 即将开始的活动：确认库存充足、价格正确
- 活动进行中：监控实时销量和排名变化
- 活动即将结束：准备承接流量和评价管理
参考：pdd-activity-calendar skill

### 8. 店群日检（店群卖家必做，v8.0新增）
- 各店IP/登录环境稳定性检查
- 跨店关联风险指标扫描（同IP段/同发货地址告警）
- 赛马机制数据快照（先锋店/主力店/清仓店各自GMV和转化率）
- 内部价格监控（避免店群内部价格战）
参考：pdd-store-matrix skill

## 每周必做（2小时）

### 周一：活动复盘 + 竞品监控
- 回顾上周各商品的活动表现：流量变化/转化率/活动ROI
- 规划本周活动报名（提前锁定秒杀/百亿补贴档期）
- 更新Excel跟踪：竞品链接 | 7日销量 | 售价 | 近30天评价数 | 差评率 | 促销活动 | 主图更新
- 关注：销量突增（可能砸钱推广）、频繁换图（测试视觉）、评论数突增但差评也突增（刷单嫌疑）

### 周三：上新
上架2-3款新品（选品配合 sourcing 管线），含全套图片+标题+详情优化

### 周五：推广复盘
回顾多多搜索/场景/全站推广数据，调整出价和预算
- 二级词ROI最优（3-5倍+），重点保留
- 场景CPC 0.1-0.3元，适合爆款后期
- 全站推广ROAS目标2.5-3.5

### 周日：差评汇总
整理本周差评和客户反馈，制定改进措施

### 供应链质检周报（直采卖家必做，v8.0新增）
- 本周到货批次自检结果（对照L1自检清单：面料/尺码/做工/色差/瑕疵）
- 退货原因分类统计（直采品质问题占比）
- 供应商交期履约率
- 需要反馈给工厂的品质问题清单
参考：pdd-c2m-supply-chain skill 第六章质检体系

## 每月必做（半天）

1. 盘点当月销售额、毛利率、退货率三大指标
2. 淘汰零销量且无转化的滞销款
3. 规划下月促销活动和上新计划（对照拼多多活动日历 → 加载 `pdd-activity-calendar` 获取完整排期和报名策略）
4. 检查DSR评分趋势，目标三项保持4.8+分
5. **平台政策回顾**（新增）：检查拼多多当月规则变更/新功能/政策调整，评估运营影响
6. **推广工具审视**：是否到了从搜索→全站推广过渡的阶段？全站推广ROI是否达标？
7. DSR分项优化：
   - 描述相符：实物和图片一致，面料厚度用尺子量
   - 物流服务：48小时内发货
   - 服务态度：响应30秒内

## DSR维护要点

| 评分项 | 优化方法 | 目标 |
|:-------|:---------|:-----|
| 描述相符 | 详细厘米数测量表、色差免责声明、质检（剪线头/检查拉链） | 4.8+ |
| 物流服务 | 48小时发货、自动提醒 | 4.8+ |
| 服务态度 | 30秒响应、快捷短语、语气亲切 | 4.8+ |
| 退货率 | 控制在15%以内，尺码不准(40%)>质量(30%)>色差(20%) | <20% |

## 店群矩阵日检（⭐ v8.0新增）

> 详细店群运营框架见 `pdd-store-matrix` 技能。

如运营多店铺，每日新增以下检查项（+15分钟）：

### 隔离环境检查
- 各店铺IP/登录环境是否稳定（指纹浏览器连接状态）
- 同一IP段登录报警（超过1个店铺直接触发隔离失败）
- 各店铺浏览器指纹是否保持独立

### 跨店运营检查
- 各店DSR独立维护（一店掉分不影响他店）
- 跨店竞品价格监控（避免内部价格战）
- 各店铺发货地址/快递公司是否差异化
- 客服话术是否使用不同版本（最少3种表述）

### 赛马机制数据
如采用"先锋店→主力店→清仓店"三级赛马机制：
- 先锋店新品数据采集（48小时点击率/转化率）
- 主力店爆款ROI跟踪
- 清仓店滞销库存预警
- 月度店铺淘汰评估

## 供应链质检日检（⭐ v8.0新增）

> 详细C2M直采体系见 `pdd-c2m-supply-chain` 技能。

如已切换1688/产业带直采，每日新增：

### 到货抽检（L1自检）
- 面料手感/异味检查
- 随机3-5件尺码测量（胸围/腰围/衣长偏差≤2cm）
- 做工快速扫描（线头/拉链/印花偏移）
- 色差对比（与样品在同一光源下）

### 退货根因追踪
- 记录每笔退货的工厂批次号
- 同一工厂/批次退货率>25% → 暂停该工厂新单
- 品质问题反馈工厂（附照片+批次号），要求改善方案
