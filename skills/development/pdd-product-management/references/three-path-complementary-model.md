# 三路径互补模型 — 拼多多全链路自动化发布方案

## 模型概述

单一自动化路径各有不可弥补的缺陷，三条路径形成互补三角：

```
         ┌──────────────────────┐
         │    🥇 API 路径       │
         │  pdd.goods.add       │
         │  100% 自动化率       │
         │  需 ISV 审核         │
         └──────────┬───────────┘
                    │
    ┌───────────────┼───────────────┐
    │               │               │
    ▼               ▼               ▼
 日常主力       API不通时       两者都失败时
 数据驱动       零门槛兜底       保留手动能力
    ▲               ▲               ▲
    │               │               │
┌───┴───────┐ ┌─────┴─────┐ ┌─────┴──────┐
│🥈 CSV批量 │ │🥉Playwright│ │  人工录入  │
│ Excel模板 │ │ 半自动模式 │ │  最终保底  │
│ 90%自动化 │ │ 40%自动化  │ │  0%自动化  │
│ 零门槛    │ │ 零门槛    │  │  100%可靠  │
└───────────┘ └───────────┘ └────────────┘
```

## 三路径详细对比

| 维度 | 🥇 API | 🥈 CSV | 🥉 Playwright |
|:------|:------|:------|:-------------|
| 自动化率 | 100% | ~90% | ~40%（半自动） |
| 门槛 | ISV审核 | 格式验证 | 零门槛 |
| 上新速度 | 千款/天 | 百款/批 | 3-5款/小时 |
| SKU支持 | 完整 | 待验证 | 需手动规格 |
| 规格处理 | API参数 | 模板展开 | 下拉模板+手动 |
| 图片处理 | URL上传 | 待验证 | 拖拽+选择器 |
| 审核机制 | 48h自动 | 可能不同 | 48h自动 |
| 合规性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| 可靠性 | 99%+ | 95%+ | 70% |
| 月成本 | ¥0-200（API调用） | ¥0 | ¥0（开发成本） |
| 维护成本 | 极低 | 低 | 高（页面变更） |

## 操作指南

### 场景路由

```
日常发布(1-3款/天) → API pdd.goods.add
批量上新(10-30款)   → CSV 批量导入
API不通/审核未过    → CSV 为主
CSV格式不支持       → Playwright 半自动
Playwright全阻塞    → 人工操作 + 记录阻塞点
```

### 风险矩阵

| 风险 | 概率 | 影响 | 应对 |
|:----|:----|:----|:-----|
| API ISV审核被拒 | 中 | 高 | 切CSV为主路径 |
| CSV格式不支持多规格 | 中 | 中 | 单规格商品走CSV，多规格走Playwright |
| Playwright页面改版 | 高 | 低 | 日常维护selector，只做基本信息 |
| beast-core升级新防御 | 中 | 中 | 有3条路径可切换，不依赖单一方案 |
| API接口废弃 | 低 | 极高 | 需PDD提前通知，有迁移时间 |

### 降级链

```
首选: API pdd.goods.add
  ↓ (ISV审核被拒 / API不通)
备选: CSV批量导入
  ↓ (模板不支持多规格 / 格式变化)
兜底: Playwright半自动 (基本信息+图片+草稿→人工补规格+提交)
  ↓ (页面全改版)
保底: 纯人工操作 (记录时间，推动API申请)
```
