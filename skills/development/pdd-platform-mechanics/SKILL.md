> 此技能是 ec-domain 所有策略决策的认知基础层。先理解平台，再做运营。

## 一、平台本质：活动驱动型，不是搜索驱动型

### 流量结构（核心认知）
拼多多的流量分配与传统电商（淘宝=搜索、京东=自营）根本不同：

| 流量来源 | 占比 | 特征 |
|:--------|:----|:-----|
| 活动流量（秒杀/百亿补贴/9.9特卖等） | 40-50% | 平台主动分配，免费/低成本 |
| 搜索流量（关键词排名） | 20-25% | 传统SEO逻辑 |
| 推荐流量（个性化推荐） | 15-20% | 「货找人」算法，对新品友好 |
| 付费流量（推广工具） | 10-15% | 需要预算 |
| 其他（关注/分享/复购） | 5-10% | 私域+社交裂变 |

**关键含义**：活动是拼多多最大流量入口。一个商品参与了活动 vs 没有参与活动，流量差距可达10-50倍。

### 流量分配核心指标：价格力
- 活动价比淘宝/竞品至少低10%以上（否则无法通过审核）
- 划线价与活动价差额需≥30%（创造「捡便宜」感知）
- 推荐系统以「商品品质、价格、商家服务」综合判定——「质价比」越高，自然流量越多
- **低价即流量**，不是「低价+付费推广才有流量」

### 「货找人」vs「人找货」
- 传统电商：用户搜索→浏览→比价→购买（人找货）
- 拼多多：推荐/活动/拼团把商品推到用户面前（货找人）
- 这意味着：主图+价格比标题SEO更重要；冲动消费场景比理性比价场景更多

## 二、搜索排名因子（精确权重）

据电商运营知识库和vzkoo研究报告（Tier 3）：

| 因子 | 权重 | 说明 |
|:----|:----|:-----|
| 销量 | 30% | **活动销量计入搜索权重** — 这是关键！活动不仅可以免费拿流量，还能提升搜索排名 |
| 转化率 | 25% | 主图+价格决定转化率 |
| 点击率 | 20% | 主图和首图是第一战场 |
| DSR评分 | 15% | 描述/服务/物流三项 |
| 复购率 | 10% | 农产品品类最明显 |

**策略含义**：
1. 先上活动拿销量→销量提升搜索排名→自然搜索流量增加→正向飞轮
2. DSR不是「保持就行」，而是直接影响15%搜索权重
3. 主图（影响CTR 20% + 转化率25%）是拼多多最重要的视觉资产

## 三、推广工具三代演进

### 演化路径（从天风证券研报，Tier 2）

```
自定义推广(第1代) ──→ OCPX智能出价(第2代) ──→ 全站推广(第3代)
手动设置出价/关键词    系统自动调整出价        一站式覆盖搜索+场景
广告单价<1元/笔        降低操作复杂度          商家仅设ROI目标
```

### 各工具定位与场景选择

| 工具 | 类型 | 何时用 | 何时不用 |
|:----|:----|:------|:--------|
| 多多搜索 | CPC关键词竞价 | 新品需要搜索排名、精准关键词有转化 | 预算有限、类目词竞价过高 |
| 多多场景 | CPM展示广告 | 爆款后期收割长尾流量、清库存 | 新品冷启动（难以带动免费流量） |
| 全站推广 | OCPX全自动 | 商品有50+好评后维稳放量、不想精细操作 | 新品没有数据积累时 |
| 多多进宝 | CPS佣金 | 达人/社群推广、零成本起步 | 利润率极薄的商品（佣金+券后可能亏损） |

### 阶段推进策略

```
第1-2周（冷启动）：活动报名拿基础销量 + 多多搜索拉关键词排名
第3-4周（爆发期）：秒杀/百亿补贴冲量 + 搜索继续投入
第5-8周（稳定期）：全站推广接管，设置ROI目标 2.5-3.5
第9周+（收割期）：多多场景低成本覆盖 + 全站推广维稳
```

**大多数商家推广费率在10%以上。** 需要在预算中预留这一块。

## 四、拼多多运营链路（与传统电商的本质区别）

### 传统电商链路（线性）
选品 → 供应链 → 上架 → 定价 → 推广 → 流量 → 转化 → 售后 → 数据 → 扩张

### 拼多多运营链路（以供应链+活动为起点）
**供应链（C2M/产业带）→ 定价（全网最低价）→ 活动报名（获取流量）→ 流量爆发 → 转化（低价+主图）→ 数据反馈 → 选品调整 → 付费推广维稳 → 售后（防差评）→ 多店铺矩阵扩张**

**关键差异**：
- 供应链前置：不是「选好品再找货」，是「有什么供应链做什么品」
- 流量先于转化优化：先通过活动拿量，再做精细化
- 评价权重极高：1个差评可以毁掉一个链接的所有投入
- 规模化靠店群矩阵而非单一品牌旗舰店

## 五、平台政策动态（2025-2026关键变化）

### 政策转向：从「极致讨好消费者」到「平衡商家权益」
- **2025年1月**：成立商家权益保护委员会（商保会），联席CEO赵佳臻亲自带队
- **2025年**：「仅退款」政策松动，1688率先取消，行业开始回调
- **2025年**：推出「千亿扶持」计划，三年投入千亿助力中小商家。核心举措包括：
  - 保证金下调（1000→500元），覆盖近70个类目
  - 退款订单自动返还推广服务费
  - 偏远地区物流中转费降幅>90%（40多元→4-5元）
  - 售后申诉取消次数限制，恶意订单申诉成功率+50%
  - 73%商家认为优化"仅退款"十分有用，82%认为权益保障提升有用
- **2025年3月**：消息称拼多多将再投千亿扶持中小商家转型升级（官方回应"正在研究"），战略重心从"消费者绝对优先"转向"商家-消费者平衡"
- **2025年**：市场监管总局明确规范「仅退款」「自动跟价」「全网最低价」规则
- **2026年趋势**：平台正在从极端偏向消费者转向更平衡，这是商家利好窗口

### 自动化工具的合规性光谱（2026-05 P0研究）

> 完整分析见: `~/research-skill-graph/projects/pdd-full-automation-feasibility/p0-compliance-research.md`

| 层级 | 路径 | 合规性 | 说明 |
|------|------|--------|------|
| 🟢 | 官方API (pdd.goods.add等) | 100%合规 | 需ISV审核（企业资质，~¥4,800/年云部署） |
| 🟡 | 服务市场ERP工具 | 合规 | 聚水潭/快麦/客优云等已审核工具 |
| 🟠 | 商家后台批量功能 | 安全 | CSV批量导入等平台原生功能 |
| 🔴 | 浏览器自动化 | 高风险 | Playwright/Selenium模拟操作，平台保留处置权 |

**关键法律事实**：
- 开发者协议赋予拼多多"基于普通非专业人员知识水平单方认定违约"的宽泛权力
- 平台可"提前15天通知解除协议而无须承担违约责任"
- 赔偿上限为零（免费使用=零责任）
- 关联处罚机制：一圈全封（同IP/设备/提现账户的店铺统一处理）
- **未找到明确禁止"自动化上架"的协议条款**，但宽泛处置权覆盖一切

### 需持续关注的监管风险
- 反垄断审查（拼多多市场份额接近20%预警线）
- 「全网最低价」规则可能被限制
- 自动跟价功能可能被调整
- 商家罚款机制可能进一步改革

## 六、竞争格局速览

| 平台 | 2024年GMV | 2024年份额 | 核心优势 | 增速趋势 |
|:----|:--------|:----------|:--------|:--------|
| 淘宝天猫 | ~8万亿 | 31% | 品牌资源、搜索生态 | ↓ |
| 拼多多 | ~5.2万亿 | 20.1% | 低价心智、C2M供应链 | ↓（增速降至10%） |
| 抖音电商 | ~3.5万亿 | 18% | 8亿DAU、内容闭环 | ↑↑（最快） |
| 京东 | ~3万亿 | 19.4% | 自营物流、3C正品 | → |

**抖音电商是拼多多最大竞对**：预计2025年GMV突破4万亿，直逼拼多多5.2万亿。抖音广告收入已超越阿里成为中国第一。拼多多不得不在2025年与抖音「牵手」搞双平台直播。

### 对商家运营的影响
- 仅聚焦PDD内部优化不够，需要建立跨平台（特别是抖音）竞争意识
- 拼多多商家正面临平台增长放缓 → 流量竞争加剧 → 推广成本上升的循环
- 视频号电商正在起量，小程序+直播+社群模式值得关注

## 七、核心概念速查

| 概念 | 含义 | 运营含义 |
|:----|:----|:--------|
| C2M | Customer-to-Manufacturer，工厂直达消费者 | 去中间商，结构性低价 |
| 全站推广 | 第3代推广工具，一站式覆盖搜索+场景 | 商品50+好评后启动 |
| 百亿补贴 | 平台出资补贴品牌商品 | 品牌化入口，非品牌商品难参加 |
| 仅退款 | 消费者无需退货即可退款 | 2025年起松动，利好商家 |
| 货找人 | 推荐算法主动推商品给用户 | 主图+价格比标题重要 |
| 店群模式 | 多店铺矩阵测试不同商品 | PDD规模化标准解法 |
| 商保会 | 商家权益保护委员会（2025年成立） | 平台态度转向信号 |
| 低价引流 | 通过设置不合规低价SKU骗取点击的违规行为，本质是价格欺诈 | 2025年8月列为开店三大禁忌之一 |
| SKU价差红线 | 同一链接SKU价差>20%或设1元SKU即触发限流的平台硬性规则 | 零容忍，违规扣保证金1000元+降权30天 |
| 双层审核 | 商品发布审核（48小时内）+ 活动报名审核的双重过滤机制 | 每次修改除库存/上下架外均需重新审核 |
| 拼单价/单买价 | 拼多多特有的双价格体系：拼团价 + 单独购买价 | 拼单价是最低展示价，也是活动价参考基准 |

## 八、商品上架与商品管理（全流程）

> 完整9透镜深度研究产出：`~/research-skill-graph/projects/pdd-product-management/`
> 本节为平台机制层的速查摘要，完整分析见研究项目4文件。

### 商品发布8步漏斗

```
1. 分类选择（一/二/三级类目）—— 选错影响审核，中老年女装选「女装→中老年女装」
2. 基本信息（标题/品牌/属性）—— 品牌可选「其它/other」，属性按类目填写
3. 规格与库存 —— 自定义或标准规格，笛卡尔积自动生成SKU表格
4. 价格设置 —— 拼单价+单买价，支持统一/批量/逐个三种模式
5. 图片上传 —— 主图/轮播图/SKU预览图/详情页/主图视频
6. 运费模板 —— 新建或选择，支持区域差异化
7. 服务承诺 —— 预售/48小时发货/7天无理由/运费险
8. 提交审核 → 审核中(48h内) → 通过/驳回
```

### SKU管理核心规则

| 规则 | 限制 | 说明 |
|:----|:----|:-----|
| 规格值上限 | 44个（T恤/沙发垫/门帘60个） | 多规格值加起来计算 |
| 规格维度上限 | 最多20个 | 单维度≤10个规格值 |
| SKU编码 | 最多100字符 | 库存量单位唯一标识 |
| 规格名称 | 最多18字 | 超限被截断 |
| SKU价差 | ≤20% | 零容忍红线，违规触发降权 |
| SKU预览图 | 等宽高≥480px，≤1MB，JPG/PNG | 必须为商品属性图 |

### 中老年女装SKU设计陷阱

- 颜色7种 × 尺码8个（S-5XL）= 56个SKU → **超出44上限**
- 解决方案：利用T恤类目60上限，或精简颜色/合并尺码
- 尺码必须覆盖XXL-5XL，加大码库存占总库存30%+

### 审核驳回常见原因（高频Top5）

1. SKU价格范围过大（价差>20%）
2. 标题量词与最低价SKU不匹配（写「一对」但最低价是「一个」）
3. 主图与最低价SKU商品不一致（展示套装但最低价是配件）
4. 图片质量不达标（模糊/水印/非白底/拼接）
5. 类目选择错误或资质不齐全

### 违规处罚阶梯（2026最新）

| 程度 | 处罚 | 
|:----|:----|
| 首次/轻微 | 搜索屏蔽7天 + 商品下架 + 扣保证金1000元 + 降权30天 |
| 中重度 | 移除资源位 + 禁止上新/上架 + 移除广告 |
| 严重欺诈 | 三倍赔偿(最低500元) + 罚款≥1000 + 冻结货款 + 封店 |

### 商品管理后台架构

- 前端：React SPA + beast-core自研组件库（InputNumber/Select/AutoComplete）
- 状态管理：Redux/Mobx
- 批量操作：支持批量上下架、批量改价、批量改库存
- 关键行为：修改库存/上下架不触发审核，其余修改均触发重新审核
