# 反应式技能固化 (Reactive Skillify)

## 触发条件

以下任一满足时**自动触发**：

1. **5+ 工具调用的复杂任务完成** → 检查是否可固化为技能
2. **用户重复纠正同一类问题** → 把纠正规则写入记忆或技能
3. **发现可复用的流程模式** → 保存为技能供下次使用
4. **用户明确说"记住"/"记下来"** → 立即执行

## 判断标准

### 适合固化为技能的场景
- ✅ 多步骤工作流（3步以上，顺序固定）
- ✅ 有明确的输入/输出
- ✅ 可在不同时间/不同上下文重复使用
- ✅ 涉及特定命令/参数组合
- ✅ 用户之前已经执行过类似操作

### 不适合固化的场景
- ❌ 一次性任务（修改某个特定参数）
- ❌ 单纯的信息查询（搜索、阅读）
- ❌ 用户个人的临时偏好
- ❌ 任务本身就是调试/排错（一次性的）

## 执行流程

```
复杂任务完成
    ↓
[触发检查] 工具调用次数 ≥ 5？
    ├── 否 → 跳过
    └── 是 → 继续
        ↓
[模式分析] 这个流程：
    ├── 步骤是否固定、可重复？ → 否 → 跳过
    ├── 是否有明确输入输出？ → 否 → 跳过
    ├── 未来是否可能再用？ → 否 → 跳过
    └── 全部是 → 执行固化
        ↓
[固化操作]
    1. 提炼步骤为 markdown 格式
    2. skill_manage(action='create') 创建技能
    3. memory 添加技能索引
    4. 在汇报末尾注明："⚡ 已固化为 [技能名] 技能，下次直接说'...'即可"
```

## 技能命名规则

```
{域}-{动作描述}
例如：
- ec-batch-download — 批量下载17网商品
- stock-pe-roe-screen — PE/ROE多因子选股
- pdd-listing-review — PDD上架数据审核
```

## 举例

### 正确例子
```
发现用户经常说"帮我选8件中老年女装裙子"→ 每次执行同样的 pipeline
→ 固化技能: ec-quick-pick
→ 下次说"帮我选裙子"即可触发完整管线
```

### 错误例子
```
用户改了一个文件中的某个变量值 → 一次性修改，不固化
```

## 规则

1. **不要问用户** — 直接分析判断，直接执行
2. **不要过度固化** — 每天最多固化1-2个技能，避免膨胀
3. **保持简洁** — 技能正文不超过200行
4. **立即通知** — 固化后在下条回复末尾标注
