# Find Skills

当用户只描述目标、但未指定具体工具或技能时，优先使用本技能做自动匹配。理解用户意图，从仓库中精准定位、安装并配置合适的技能。

## 工作方式

1. **语义理解**：根据模糊目标生成 3-5 个核心技能检索词
2. **多源检索**：在仓库、本地已安装列表及相关索引中搜索候选
3. **优先级排序**：按评分、下载量、更新频率、匹配度综合排序
4. **决策与理由**：为前 3 名给出详细推荐理由
5. **按需执行**：确认后执行安装

## 触发条件

- **冷启动**：新项目，Agent 之前没处理过的任务类型
- **工具链补强**：现有技能集无法完成任务
- **优化替代**：现有技能效率低下，找增强版
- **批量部署**：一键安装全套开发/运维技能

## 核心能力

### 1. 精准意图映射
- 对 Prompt 做"原子任务"拆解
- 映射为标准技能标签
- 检索词应为"动词+名词"结构，如 `parse-xlsx`

### 2. 候选技能对比评估
- 提取候选技能 description 和 metadata
- 建立评分矩阵（功能覆盖度/稳定性/易用性）
- 优先推荐 verified 标签或生产环境验证过的

### 3. 一键环境就绪
- 安装后自动检查是否需要环境变量或 OAuth
- 运行 hello-world 级别测试确保可用

### 4. 依赖项解析
- 自动安装底层依赖（puppeteer/ffmpeg 等）

## 推荐模板

```
### 用户目标
[描述]

### 识别关键词
- keyword1
- keyword2

### 推荐技能组合
1. skill-a (评分 4.8): 理由
2. skill-b (评分 4.5): 理由

### 是否安装？
> [Yes] / [No] / [Tell me more]
```

## 边界

- 受限容器或无 root 权限环境可能安装失败
- 防止安装功能冲突的多个技能
- 非官方技能给出安全警告
- 老旧技能可能在当前环境无法运行

## 外部大规模技能库评估（精而不杂）

当面对 200+ 技能的外部集合（如 claude-skills）时，用 3 轮过滤代替逐个审查：

### 第 1 轮：域无关性过滤
逐域对照用户系统，排除完全不相关的域：
- 医疗器械合规/ISO/FDA → 排除
- K8s/Helm/Terraform（单机部署）→ 排除
- C-suite 顾问/Jira/Confluence → 排除
- 判断标准：如果该域在用户未来 6 个月内 100% 用不到 → 排除

### 第 2 轮：重叠去重
将剩余技能与已安装技能做逐项对比：
- 已有 Superpowers 14 技能 → 排除工程流程重复项
- 已有 100+ 精选技能 → 排除同功能技能
- 判断标准：功能覆盖度 > 80% 或已安装版本更优 → 排除

### 第 3 轮：ROI 精筛
仅保留填补明确空白或显著增强现有能力的技能：
- 填补空白（优先级最高）：系统完全缺失的能力
- 显著增强（优先级中等）：补充现有域的薄弱环节
- 判断标准：能否在 3 句话内向用户解释"为什么装这个而不是已有 X"

### 反模式
- ❌ "这个看起来有用" — 有用不等于值得装
- ❌ "全装再看" — 技能不是越多越好，精而不杂
- ❌ "用户可能会用到" — 只装现在或近期确定需要的
