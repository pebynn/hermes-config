# MEMORY 瘦身三步法 (2026-05-13 实战)

## 目标
MEMORY usage < 70%，entry_count ≤ 22。每轮省1000+ token。

## 三步流程

### Pass 1: 删 stale（已过时/已修复/已退役）
识别信号：
- 修复记录中写了具体日期且>7天前的bug
- 已退役组件的描述（Hindsight, delegate_task, iLink等）
- "已修复"/"已删除"/"已于X日" 等完成标记

操作: `memory remove` — 直接删除，不回看。

**本次**: 删5条 (修复记录3 + 退役组件1 + 已修复1) → 省 ~500 字

### Pass 2: 删 overlap（与 SOUL.md/lessons 重复）
识别信号：
- 规则条目在 SOUL.md 有对应章节（路由铁律、决策矩阵、诊断铁律）
- 教训条目在 lessons/global.md 有对应 🔴CRITICAL
- 架构描述在 SOUL.md 全局架构图中已覆盖

操作: `memory remove` — 直接删除，加引用指针到源文件。

**本次**: 删6条 (kanban架构、路由铁律、行为约束、铁律-内容、B/D层进化) → 省 ~800 字

### Pass 3: 压缩 verbose（>150字的条目）
识别信号：
- 单条目超过150字
- 含具体脚本路径/命令/日期细节

操作: `memory replace` — 保留核心信息，删细节，加引用指针。

**本次**: 压缩3条 (铁律 92→48字, 教训 178→56字, 量化策略 215→113字) → 省 ~200 字
