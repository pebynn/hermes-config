# 回测延迟买入模式: 今日因子→明日开盘

## 问题

回测中常见的"昨日因子→今日开盘买"虽然无前视偏差，但与实盘"今日收盘因子→明日开盘买"的时序不对齐。若要改用今日因子，直接买在今日开盘存在前视偏差（今日开盘时今日因子未知）。

## 方案: pending_buys 延迟执行

在回测循环中加入一个 `pending_buys` 列表，买入决策不在当天执行，而是defer到次日：

```
Day cd (调仓日):
  1. 执行昨日pending_buys → pos (次日开盘价买入)
  2. 用 cd 截面计算排名 (xr = panel.xs(cd))
  3. 卖出逻辑 (pos中所有持仓, T+1保护当日买入)
  4. 买入决策 → pending_buys (不直接加pos)

Day cd+1:
  1. pending_buys 执行 → pos (以cd+1开盘价买入)
```

## 代码改动

### 1. 初始化
```python
pending_buys = []   # 买入信号, 延迟到次日开盘执行
```

### 2. 每日循环开头: 执行延迟买入
放在 pending_exits 执行之后、xr 赋值之前：

```python
# ── Execute pending buys from previous rebalance day ──
for c in list(pending_buys):
    if c in xt.index:          # xt = 今日截面
        ep = float(xt.loc[c, "开盘"])
        if ep > 0:
            pos[c] = {"ep": ep, "in": cd, "pc": ep, "high_close": ep}
pending_buys.clear()
```

### 3. xr 改用当日截面
```python
# 旧: xr = panel.xs(dates[ci-1])  # 前一日截面
# 新:
try:
    xr = panel.xs(cd).copy()       # 当日截面 (用于排名)
except (KeyError, Exception):
    xr = None
```

### 4. 买入段: populate pending_buys 而非 nc
```python
# 旧: nc[c] = {"ep": ep, "in": cd, ...}
# 新:
pending_buys.append(c)
```

## T+1 保护

当日买入的持仓 (`p["in"] == cd`) 被卖出段的 T+1 检查自然保护：
```python
if p["in"] == cd:      # 当日买入, 不能卖
    nc[c] = p
    continue
```
后续所有停止损/排名掉落/组合止损逻辑均被跳过。

## 为什么正确

- **无前视偏差**: 排名用 cd 截面（收盘后可知），执行用 cd+1 开盘价（在回测中已记录但未用于决策）
- **与实盘100%对齐**: 实盘 cron 16:08 读收盘数据排名 → 次日9:25挂单
- **T+1合规**: pending_buys 买入的持仓当日不可卖

## 性能影响

无明显性能影响。`panel.xs(cd)` 与 `panel.xs(dates[ci-1])` 耗时相同。pending_buys 列表通常≤5项，逐日遍历开销可忽略。

## 适用场景

任何基于因子排名的回测，需要"当日因子→次日执行"以避免假前视偏差。尤其适用于：
- 与实盘信号生成器共用的因子/排名函数
- 需要验证信号生成器输出和回测输出一致性的场景

## ⚠️ 致命陷阱: 延迟买入在市场年份间表现可天差地别

延迟1天执行在不同市场环境下的效果完全不同，必须双测。策略A v2完整实测矩阵：

> 当前实盘配置 = #4（全市场-688/92-ST + 次日开盘 + pending_buys）。信号生成器已对齐。

**关键洞察**:
- **延迟效应**: 2025年延迟吞80%alpha(50.7%→9.9%), 2026年延迟多吃176%年化(1202%→1378%)
- **池子浓缩放大不稳定性**: 主板池2025年-18.2% (全市场还9.9%), 2026年1638% (全市场1378%)
- **市值过滤反效果**: 剔<50亿小票→年化腰斩(1378%→743%), 小票是2026动量主力
- **整体结论**: 全市场-688/92 是两年都正配置的唯一选项；池子越浓缩年份间差异越大

**根因**: 
- **2025 动量衰减极快** — 今日排名Top的票次日开盘涨幅已跑完，1天gap = alpha全失
- **2026 动量持续性强** — 强票次日还有跳空高开，延迟反而多赚

**规则**: 
1. pipeline搭建时，先跑当日收盘立即执行回测作为baseline
2. 再加pending_buys对比，看延迟是加分还是减分
3. 两版结果差距>3x → 策略对执行时机极度敏感 → 该策略不适合实盘（实盘无法在当日收盘价买入）
4. 若延迟版更优 → 说明动量持续性强，实盘有信心；若退化严重 → 策略信号衰减太快
