# v4.0 stock-sdk 资金流集成 (2026-05-08)

## 架构变更

v3 → v4.0:
- L4(两融)移除 → 替换为stock-sdk资金流
- 资金流升级为独立维度,权重0.20(仅次于缠论0.30)
- 共振乘数机制移除,改直接加法合成
- scan_signals从顺序for→4路multiprocessing.Pool并行
- policy_detect.py(87min瓶颈)移除,硬编码policy_multiplier=1.0

```
composite = (L1_scaled×0.25 + L2_score×0.30 + ff_score×0.20 + L3_total×0.25) × nb×policy
```

## 并行扫描

```python
# signal_engine.py scan_signals()
from multiprocessing import Pool
with Pool(4) as pool:
    for row in pool.imap_unordered(_scan_worker, args_list, chunksize=10):
        if row: results.append(row)
```

预期吞吐: 4 worker × 1只/s = 4只/s, 全量~30s (vs 旧顺序2-3min)

## 资金流评分

`stock_fund_flow.py` 评分函数:
- `score_main_net(净流入额, 成交额)` → 0-100 (权重40%)
- `score_main_net_ratio(占比%)` → 0-100 (权重35%)
- `score_retail_net(散户净流入, 成交额)` → 0-100 反向指标 (权重25%)

## 盘后资金流不可用策略

stock-sdk fund flow依赖东方财富push2, 盘后(19:00-08:00)返回全0。
**解决方案**: 14:45预采集cron → 缓存到 `~/.finquant/cache/fund_flow/{date}.parquet`
scan_signals时从缓存读取,无数据默认ff_score=50(中性)。

## 坑点

- fund flow API 22:30返回全0 → 不是bug,是盘后无数据
- stock-sdk getFundFlow 返回的code字段可能为"1" → 需过滤 `code != "1"`
- NODE_PATH需指向 stock-sdk-mcp/node_modules 才能import stock-sdk
- share_db.parquet缺失 → turnover算不出 → 按流水账补 `load_share_db()`
