# v4.0 scan_signals 并行化 + 挂死点消除 (2026-05-08)

## 改动内容

### 1. scan_signals 顺序→multiprocessing.Pool(4)

```python
# 改前: 顺序for循环
for i, code in enumerate(codes):
    row = _compute_one_stock(code, kline_dir, industry, share_db, ...)

# 改后: 4路Pool并行
from multiprocessing import Pool
with Pool(4) as pool:
    args_list = [(code, industry_map.get(code,""), kline_dir, share_db, start_date, mc_min, mc_max) for code in codes]
    for i, row in enumerate(pool.imap_unordered(_scan_worker, args_list, chunksize=10)):
        ...
```

**必须**: `_scan_worker` 必须是 module-level 函数(非闭包/嵌套), 否则 multiprocessing pickling 失败。

### 2. policy_detect 整块移除

`signal_engine.py` 中原有 `detect_policy()` 调用 → 13步央视新闻联播全文检索, 每步~400s, 总计~87min。v4.0 删掉 import + try/except 块, 替换为 `policy_multiplier = 1.0` 常量。保留变量名供输出列引用。

### 3. 北向资金AKShare晚间跳过

AKShare `stock_hsgt_fund_flow_summary_em()` 在北京时间19:00-08:00调用会阻塞60-120s直到系统socket超时。加时间窗口判断:

```python
import datetime
_now_h = datetime.datetime.now().hour
if not (_now_h >= 19 or _now_h <= 8):
    # call AKShare
else:
    northbound_net = 0.0  # skip
```

### 4. 行业中性化 duplicate index 防护

并行返回的结果可能有重复code, 导致 `df_z.loc[code, fk]` 返回 Series 而非 scalar:

```python
df = pd.DataFrame(rows, index=codes)
df = df[~df.index.duplicated(keep='first')]  # 新增
```
