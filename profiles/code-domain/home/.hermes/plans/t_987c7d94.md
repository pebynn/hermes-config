## 2. Design Decisions

### 2.1 Stop Loss: 5% → 8% (HIGH IMPACT)
**方案**: 放宽个股止损从5%到8%，给持仓更多呼吸空间。
**替代方案**: 阶梯式止损(5%→7%→9%按持仓天数)，但增加参数复杂度。
**风险**: 放宽止损可能增加单笔亏损幅度。8%已经接近当前non-stop组合的平均亏损基线，是合理平衡点。
**预期**: 止损率从62.5%降至40-50%，WR从27.8%升至35-45%。

### 2.2 Trailing Stop: 新增移动止盈 (MEDIUM IMPACT)
**方案**: 从entry后跟踪最高收盘价，从高点回撤>8%时止盈退出。
**替代方案**: 固定止盈(+15%)，但会截断大赢家。
**风险**: 可能在震荡中过早退出趋势股。8%的trailing阈值需与个股止损协调。
**实现**: 在pos dict中新增`high_close`字段追踪，退出时记录reason="trail"。
**预期**: 保护已有利润，减少"从大赚变止损"的案例。

### 2.3 Factor Weights: 等权 → 动态IC权重 (MEDIUM IMPACT)
**方案**: 使用20日滚动RankIC动态调整三个因子的权重。
  - 每20个交易日计算每个因子的RankIC
  - 权重 = softmax(IC * temperature)，temperature=2避免极端权重
  - 与固定权重(0.50/0.30/0.20)形成对比
**替代方案**: 等权(当前)、PCA降维。动态IC更直接响应市场变化。
**风险**: IC计算有look-ahead bias风险(需要未来收益)。解决方案：用已实现的5日forward return计算IC，确保无未来信息泄露。
**实现**: 在ranking步骤前计算滚动IC，实时调整flow/mom/turn权重。

### 2.4 Factor Enhancement: 添加低波动因子 (LOW-MEDIUM IMPACT)
**方案**: 新增volatility因子(20日波动率，低波动优先)。
  - vol_20d = std(log_return, 20)
  - 因子值 = -vol_20d (低波动得分高)
  - 加入composite_score，权重约0.15-0.20
**替代方案**: 成交量因子(volume breakout)。波动率因子更直接解决"假突破后高波动杀跌"问题。
**风险**: 低波动因子在牛市中可能拖累收益。动态IC权重可自然降低低效因子的影响。
**预期**: 过滤掉高波动假突破标的，提高信号质量。

### 2.5 Late-Day Confirmation: 跳过 (DATA CONSTRAINT)
**方案**: 原计划14:50主力资金方向确认，但当前使用日线数据，无法实现盘中确认。
**替代方案**: 使用次日开盘价替代late-day确认（等价于当日收盘信号+次日开盘执行）。
**风险**: 此方向在当前数据约束下不可行，跳过。剩余4个改进方向已经足够。

### 2.6 Parameter Budget (≤5)
改进后的可调参数：
1. `stop_loss` — 个股止损线 (0.05→0.08)
2. `trailing_stop` — 移动止盈回撤阈值 (新增, 0.08)
3. `ic_window` — IC计算窗口天数 (新增, 20)
4. `vol_weight` — 波动率因子权重 (新增, 0.15)
5. (保留一个空位)

固定不变: TOP_N=10, LEVERAGE=1.0, PORTFOLIO_STOP=0.15, JAN_APR_EMPTY=True, TC_COST=0.001
