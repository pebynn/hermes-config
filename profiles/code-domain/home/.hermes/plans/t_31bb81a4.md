## 关键实现细节

### P0: 量比预公告化 (find_event_v4 核心改动)
```python
# OLD (FUTURE LEAK): post_vols from ann_idx+1 to ann_idx+6
# NEW (LEGITIMATE): pre_vols_recent from ann_idx-5 to ann_idx-1
#                  pre_vols_baseline from ann_idx-20 to ann_idx-6
#                  pre_vol_buildup = mean(recent) / mean(baseline)
```
- 如果 `ann_idx - 20 < 0` 或 volume 数据不足 → 返回 None
- 量比阈值保持 1.0（近期量 > 基线量 → 有资金提前布局）
- 函数重命名: `MIN_POST_VOLUME_RATIO` → `MIN_PRE_VOLUME_BUILDUP`
- 返回字段: `vol_ratio` → `pre_vol_buildup`

### P2: 重入+评分排序 (run_backtest 改动)
- 移除 `if code not in positions` 限制
- 每日开仓前: 对当天events按 score 排序 (SUE*0.4 + pre_5d_ret*0.3 + pre_vol_buildup*0.3)
- 按排序取 top N = MAX_POSITIONS - len(positions)
- hold_days 种子: `hash(code + str(event_date)) % 2**31` (每事件不同)

### P3: 移动止盈
- 新常量: `TRAIL_STOP = -0.06`
- 持仓跟踪: `peak_price` = max(entry_price, daily_high)
- Exit触发: `(price - peak_price) / peak_price <= TRAIL_STOP`
- 仅在 `hold_days_actual >= 2` 后激活
