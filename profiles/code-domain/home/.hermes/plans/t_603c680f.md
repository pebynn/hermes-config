## 背景

R12-B 表现退步（年化8.50% vs R11 10.47%），根本原因是4项R10-R12积累性偏离：
- rebalance_close出血: 252笔交易，avg -70.15/笔
- hybrid市场择时: Mar 2026与ma_cross差-2.27pp
- MOM_20D_MIN 6%→5%过宽: 新标的净亏损
- SECTOR_MAX_POSITIONS 4→3过松: 行业集中度↑
- MAX_CAPITAL_PER_STOCK 500K虚高

R14-B基准: 回归R11-B配置 (年化10.47%, 夏普2.896) + 8项针对性优化。

## 设计方案

### 方案选择: 基于R11-B的增量优化

选择理由:
- R11-B已验证: 10.47%年化, 50.9%胜率, 夏普2.896
- R12-B的4项改变全部负收益 → 全部回退
- R13-B回退到R9-B过激(年化可能更低) → 应采用R11-B为基准
- 新增4项精准优化针对R12失败根因

替代方案: 基于R9-B + 更多探索（R13-B路径）→ 已否决，因为R9-B权重[0.40,0.30,0.30,0.0]丢失R10引入的动量共振收益(+0.20 mom weight)

### 风险分析

| 风险 | 等级 | 缓解 |
|:--|:--|:--|
| TRAILING -3%过紧导致提前止盈 | 中 | R12 trailing_stop avg +694.58是最佳退出方式，-3%仅比-4%紧1pp |

## 实施细节

### 变更1: TRAILING_STOP_PCT -0.04 → -0.03
- 位置: config区 ~line 107
- 理由: R12 trailing_stop是唯一正收益退出方式(avg +694.58)
- 从-4%收紧到-3%提前锁利

### 变更2: VOL_RATIO_MIN 1.0 → 1.5
- 位置: config区 ~line 97 → 改为1.5
- 理由: 从boolean量能确认升级为1.5x均量强化筛选
- 减少无量反弹假信号

### 变更3: 新增 QUALITY_SCORE 过滤器
- 新增函数: quality_score_ok(roe, revenue_growth)
  - ROE > 0 (排除亏损公司)
  - revenue_growth > 0 (排除营收下滑)
  - NaN → graceful pass
- 新增配置: QUALITY_FILTER_ENABLED = True
- 新增配置: ROE_MIN = 0, REVENUE_GROWTH_MIN = 0
- 数据加载: 扩展fundamental_map从{pe, pb}到{pe, pb, roe, rev_growth}
- 过滤器位置: select_signals中PE/PB过滤之后(Step 7b)

### 变更4: REBALANCE_CLOSE 评分衰减机制
- 入口存储: execute_entry时保存entry_abs_ret5d = abs(信号的ret_5d)
- 每日检查: 在stop-loss/TP循环中(跑在所有退出检查之前)
  - 对每个持仓计算 current_abs_ret5d = abs(当日ret_5d)
  - 若 current_abs_ret5d < entry_abs_ret5d * 0.60 → 评分衰减退出
  - 退出原因: "score_decay"
- 关键设计决策: 使用abs(ret_5d)而非composite z-score作为评分代理
  - z-score需要候选池上下文(仅在rebalance时可用)
  - abs(ret_5d)是反转吸引力的直接度量
  - 每日循环中可直接从day_map获取
  - 60%阈值等价比率: abs(ret_5d)从5%衰减到3% = 吸引力减半
