# R3-B Strategy Reversal 优化计划

## 任务
从R2(22笔交易/3%年化)恢复到R1级别交易量(2000+笔)，同时通过信号排名优化提升年化到300%+

## 设计方案

### 方案选择: "宽池精排" (Wide Pool, Smart Ranking)

**核心思想**: R2用层层硬过滤杀死了候选池 → R3恢复R1宽池，用排名(ranking)替代过滤(filtering)

**具体参数变更**:

| 参数 | R2(当前) | R3(目标) | 理由 |
|:--|:--|:--|:--|
| DECLINE_PREFILTER | -0.07 | -0.03 | 恢复R1宽松预过滤(-3%即可) |
| DECLINE_QUANTILE | 0.05 | 0.18 | R1的18%底部分位数(不是5%) |
| MAX_POSITIONS | 8 | 15 | R1的15持仓，更多交易机会 |
| REBALANCE_DAYS | 7 | 3 | R1的3天周期，更快轮动 |
| SECTOR_HEAT_HARD_FILTER | 0.65 | 0.50 | 恢复R1宽松行业过滤 |
| RSI gate | <35(硬过滤) | 移除硬过滤，加入评分 | 不再用RSI卡候选池 |
| VOL_RATIO gate | >1.5(硬过滤) | 移除硬过滤 | 不再用vol卡候选池 |

**保留R2的优胜特性**:
| 参数 | 保留值 | 理由 |
|:--|:--|:--|
| TAKE_PROFIT | 0.20 | R2验证:让盈利跑(均+1384/笔) |
| TRAILING_STOP_PCT | -0.05 | R2验证最优(均+2558/笔) |
| HOLD_WINNERS | True | 滚仓复利 |
| WINNER_THRESHOLD | 0.03 | 仅保留≥+3%赢家 |
| STOP_LOSS | -0.08 | 保持 |
| TIME_STOP_DAYS | 5 | R2优化值 |
| TIME_STOP_RET_THRESHOLD | -0.05 | R2优化值 |
| LEVERAGE | 1.0 | 铁律 |

**信号质量优化(不卡池)**:
- 评分公式: `score = z_score(abs(ret_5d)) * sector_boost` 
  - 在候选池内计算ret_5d的z-score排名(越大越极端)
  - 乘以sector_boost做行业加权
  - 选TOP N — 宽池+精排=信号质量提升但不减量

**R2硬过滤→R3软排名对照**:
```
R2: ret_5d<-7% → RSI<35 → vol>1.5 → bottom5% → sector>=0.65 → score排序选8
     ↓ 层层卡死，最后只剩几个候选
     
R3: ret_5d<-3% → sector>=0.50 → z_score排序+score加权 → 选TOP15
     ↓ 宽池→信号排名→精选15只
```

### 替代方案
1. **渐进放宽**: 只改prefilter到-0.03，其他R2参数保持 → 可能仍不足2000笔
2. **z-score主信号**(round2_B_code方案): z_ret5d<-2.0 + 流动性过滤 → 引入新复杂度，曾导致R2→R1退步
3. **全回退R1**: 放弃所有R2改进 → 丢失take_profit/trailing的已验证提升

选择方案"宽池精排"因为它精确匹配任务要求:恢复R1交易量+保留R2信号质量改进

### 风险
- 3天周期可能仍嫌短(A股T+1)，但R1验证2051笔可行
- sector_heat数据可能缺失(仅20/5331股有行业映射)，回退到1.0中性
- z_score排名在候选池内可能不够区分度 → 回退到abs(ret_5d)纯排序

### 预期指标
- 交易笔数: ~1500-2500 (目标恢复到R1水平)
- 年化: 目标300%+ (依赖R2退出机制+宽池精排)
- 胜率: 45-55% (R1水平)
