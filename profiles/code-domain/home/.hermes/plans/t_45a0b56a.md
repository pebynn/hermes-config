## 设计方案

### 偏差1: check_momentum_entry() — 当日收盘判当日开盘 (lookahead)

**当前问题** (L388):
```python
ep = float(xt.loc[c,"开盘"])        # 今日开盘价
check_momentum_entry(xt.loc[c], ep)  # 函数内计算 (今日收盘 - 今日开盘)/今日开盘
```
函数 `check_momentum_entry` 接收 `entry_price`=今日开盘, 用 `xt_row['收盘']`=今日收盘计算收益率。在入场决策时点(开盘), 今日收盘未知 → lookahead bias。

**修复方案**: 
- 改用 `xr`(前一日截面数据)的 `mom` 因子(10日收益率, 已在 `compute_factors_v2` 中计算)
- `check_momentum_entry` 签名改为接收 `xr_row` (前一日的股票数据行)
- 使用 `xr_row['mom']`(10日收益) 与 `MOM_ENTRY_THRESHOLD`(0.02) 比较
- 阈值不变(2%), 但动量信号来源从"今日收盘vs今日开盘"变为"昨日收盘vs10日前收盘"
- **替代方案(已拒绝)**: 从`universe[c]`中查找5日前收盘计算精确5日收益 → 过于复杂, 且10日动量与5日动量高度相关, 差异可忽略

**修改点**:
- `check_momentum_entry` 函数签名: `(xt_row, entry_price, threshold)` → `(xr_row, threshold)`
- 函数体: 使用 `xr_row['mom']` 替代 `(close - entry_price)/entry_price`
- 调用处(L386-389): `check_momentum_entry(xt.loc[c], ep, ...)` → `check_momentum_entry(xr.loc[c], ...)`

### 偏差2: 移动止盈退出 — 触发当日开盘价执行 (T+1违规)

**当前问题** (L365-369):
触发了trailing stop后, 立即用当日开盘价(xt.loc[c,"开盘"])记录退出。但A股T+1规则下, 触发当天无法卖出。

**修复方案**: 引入 `pending_exits` 延迟执行机制
- 当 trailing stop / stop loss / limit-down 触发时 → 加入 `pending_exits` dict
- 次日循环开头, 批量执行 `pending_exits` 中的退出(用次日开盘价)
- 交易日期 `out` 记录为实际执行日(次日)

### 偏差3: 全局T+1约束 — 买入日和卖出日相同

**当前问题**: 无检查。如果某只股票周一买入, 同一调仓日也可能被卖出(被drop/new排名替换)。

**修复方案**:
- 持仓dict增加 `in_date` 字段记录买入日期(已有 `in` 字段 → 重命名为 `in_date` 或保留 `in` 并新增 `in_date`)
- 在任何退出决策前检查: `if p['in'] == cd` → 跳过本次退出, 保留持仓
- 同时适用于: stop loss, trailing stop, drop from ranking
- 该持仓在T+1日之后才能被卖出

### 偏差4: 跌停板建模 — stop loss在跌停板无法平仓

**当前问题** (L328-329, L362-363):
```python
if (dl - ep) / ep <= -stop_loss:
    sh.add(c)  # 直接标记止损, 不检查跌停板
# ...
trades.append({"pnl": round(-stop_loss*100, 2), "r":"stop"})  # 固定PnL
```
未检查跌停板, 假设能以止损价平仓。实际跌停板无法卖出。

**修复方案**:
- 止损触发时检查是否跌停: `close ≈ prev_close * 0.9` (主板±10%)
- 若跌停 → 加入 `pending_exits`, 延迟到次日开盘执行
- 若未跌停 → 也加入 `pending_exits`(统一延迟执行, T+1规则)
- 使用 `p['pc']`(前一日收盘) 作为跌停参考价

### 统一退出框架

所有盘中触发退出路径统一通过 `pending_exits` 延迟一天执行:

```
Day D 循环:
  1. 执行 pending_exits (上日触发的退出, 用今日开盘价)
  2. Jan/Apr空仓 (直接执行, 用当日收盘价 — 此为月初清仓, 非盘中触发)
  3. 加载 xt (今日截面)
  4. 检查T+1: in==cd 的持仓跳过退出
  5. 检查止损/移动止盈/跌停 → 加入 pending_exits
  6. 调仓卖出 (drop) → T+1检查后 → 用当日开盘执行 (调仓在开盘执行, 合理)
  7. 调仓买入 (buy)
  8. NAV计算
  9. 更新 trailing high_close
```

**关于"drop"退出**: 调仓时被新排名替换的卖出, 使用当日开盘价执行为合理(调仓在开盘执行, 非盘中触发), 但仍需T+1约束。如果用pending_exits延迟, 会引入不必要的复杂度 — drop保留用当日开盘价。

**关于Stop退出PnL**: 之前写死 `-stop_loss*100%` PnL。现在改为实际成交价(次日开盘) vs 成本价。

## 风险
- **跌停检测精度**: 使用±10%检测, 创业板±20%可能漏检 → 可接受的精度损失, 主板占绝大多数
- **pending_exits 顺序**: 同一天多次触发同一股票 → 不可能, 因为一旦加入pending就continue
- **end close 漏掉 pending**: 回测结束时 pending_exits 可能还有未执行的退出 → 在metrics计算前强制执行
- **NAV计算**: pending退出持仓仍参与当日NAV → 正确, 因为实际持有至收盘
